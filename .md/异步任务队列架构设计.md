## 一、概述

### 1.1 设计目标

Nano Banana Canvas 作为 AI 工作流编排平台，核心任务之一是处理耗时较长的 AI 生成任务。不同类型的任务有显著不同的响应时间特征：

| 任务类型       | 典型耗时  | 示例                                  |
| -------------- | --------- | ------------------------------------- |
| 视频生成       | 2-10 分钟 | Kling 视频生成、Wan 文生视频/首帧尾帧 |
| 复杂图片生成   | 30s-2min  | QWEN 图片生成、Z-Image、高分辨率 FLUX |
| 音频生成 (TTS) | 10-60s    | 长文本语音合成                        |
| 简单图片生成   | 5-30s     | 低分辨率 SD/FLUX                      |

这些任务的共同特征：**无法在单次 HTTP 请求中完成**。因此需要一套完整的异步任务处理系统。

### 1.2 为什么需要异步

**技术约束**：

- Cloudflare Workers CPU 时间限制：Free 计划 10ms，Paid 计划 30s（非 CPU 等待不计入）
- 即使使用 Paid 计划，30s CPU 时间不足以覆盖视频生成任务的完整生命周期
- 外部 AI API（Kling、Wan 等）本身就是异步设计——提交任务后需轮询获取结果

**用户体验**：

- 长时间等待同步响应会导致浏览器超时、连接中断
- 用户应能在提交任务后继续编辑其他节点
- 任务进度需实时可见（进度条、百分比）

**系统可靠性**：

- 异步解耦允许独立的重试机制
- 失败任务不阻塞后续任务
- 支持优雅降级（高峰期排队而非拒绝）

### 2.1 状态机定义

```
    ┌─────────┐     ┌─────────┐     ┌──────────┐       │

                        ┌───────────┐       ┌────────┐ │
                        │ completed │       │ failed │ │
                        └───────────┘       └───┬────┘ │
                                                │      │
                                          ┌─────┴────┐ │
                                          ▼          ▼ │
                                   ┌──────────┐  ┌─────────────┐
                                   │  retry   │──┘ dead_letter │
                                   └──────────┘   └─────────────┘
```

### 2.2 状态定义

| 状态          | 描述                       | 存储位置      | 进入条件                          |
| ------------- | -------------------------- | ------------- | --------------------------------- |
| `pending`     | 任务已创建，尚未入队       | D1            | 用户提交任务，通过积分和并发检查  |
| `queued`      | 已发送到 Cloudflare Queues | D1            | `queue.send()` 成功               |
| `running`     | Consumer Worker 正在处理   | D1            | Worker 从队列消费到该消息         |
| `completed`   | 任务成功完成，结果已存储   | D1 + R2       | AI API 返回成功 + 文件已上传到 R2 |
| `failed`      | 当次执行失败               | D1            | AI API 报错 / 超时 / 异常         |
| `retry`       | 等待重试（临时状态）       | 内存          | 失败次数 < maxRetries             |
| `dead_letter` | 彻底失败，已放弃           | D1 + 死信队列 | 失败次数 >= maxRetries            |

## 三、Cloudflare Queues 集成

### 3.1 架构角色划分

| 角色               | 实现                   | 职责                                               |
| ------------------ | ---------------------- | -------------------------------------------------- |
| Producer（生产者） | Next.js API Route      | 验证请求、扣减积分、创建任务记录、发送消息到 Queue |
| Queue（队列）      | Cloudflare Queues      | 消息持久化、至少一次交付、重试调度                 |
| Consumer（消费者） | 独立 Cloudflare Worker | 消费消息、调用外部 AI API、轮询状态、存储结果      |

### 3.3 生产者实现（Next.js API Route）

```typescript
// ============================================================
//  POST /api/tasks - 提交异步任务
//  app/api/tasks/route.ts
// ============================================================

import { nanoid } from 'nanoid'

export async function POST(request: Request) {
  const { userId } = auth()
  if (!userId) return Response.json({ error: 'Unauthorized' }, { status: 401 })

  const body = await request.json()
  const { type, provider, modelId, input, workflowId, nodeId } = body

  // ----------------------------------------------------------
  // Step 1: 并发限制检查
  // ----------------------------------------------------------
  const runningCount = await db
    .prepare(
      'SELECT COUNT(*) as count FROM async_tasks WHERE user_id = ? AND status IN (?, ?)',
    )
    .bind(userId, 'queued', 'running')
    .first<{ count: number }>()

  const maxConcurrent = await getUserConcurrentLimit(userId)
  if (runningCount && runningCount.count >= maxConcurrent) {
    return Response.json(
      {
        error: 'CONCURRENT_LIMIT_EXCEEDED',
        message: `最多同时执行 ${maxConcurrent} 个任务，请等待当前任务完成`,
      },
      { status: 429 },
    )
  }

  // ----------------------------------------------------------
  // Step 2: 积分扣减（先扣后调，失败退还）
  // ----------------------------------------------------------
  const creditsRequired = await getModelCredits(provider, modelId)
  const deductResult = await deductCredits(userId, creditsRequired, 'ai_call')
  if (!deductResult.success) {
    return Response.json(
      {
        error: 'INSUFFICIENT_CREDITS',
        message: `积分不足，需要 ${creditsRequired}，当前余额 ${deductResult.balance}`,
      },
      { status: 402 },
    )
  }

  // ----------------------------------------------------------
  // Step 3: 创建任务记录 (D1)
  // ----------------------------------------------------------
  const taskId = nanoid()
  const now = Date.now()

  await db
    .prepare(
      `
    INSERT INTO async_tasks (
      id, user_id, workflow_id, node_id, type, status,
      provider, model_id, input_data, credits_charged, created_at
    ) VALUES (?, ?, ?, ?, ?, 'pending', ?, ?, ?, ?, ?)
  `,
    )
    .bind(
      taskId,
      userId,
      workflowId ?? null,
      nodeId ?? null,
      type,
      provider,
      modelId,
      JSON.stringify(input),
      creditsRequired,
      now,
    )
    .run()

  // ----------------------------------------------------------
  // Step 4: 发送消息到 Queue
  // ----------------------------------------------------------
  const message: TaskMessage = {
    taskId,
    userId,
    type,
    provider,
    modelId,
    input,
    createdAt: now,
    retryCount: 0,
    creditsCharged: creditsRequired,
  }

  await env.AI_TASK_QUEUE.send(message)

  // ----------------------------------------------------------
  // Step 5: 更新状态为 queued
  // ----------------------------------------------------------
  await db
    .prepare('UPDATE async_tasks SET status = ? WHERE id = ?')
    .bind('queued', taskId)
    .run()

  return Response.json({ taskId, status: 'queued' }, { status: 201 })
}
```

### 3.4 消费者实现（独立 Worker）

```typescript
// ============================================================
//  Task Consumer Worker
//  workers/task-consumer/src/index.ts
// ============================================================

import { ProcessorRegistry } from './processors/registry'

export default {
  async queue(batch: MessageBatch<TaskMessage>, env: Env): Promise<void> {
    const registry = new ProcessorRegistry(env)

    for (const message of batch.messages) {
      const task = message.body

      try {
        // 更新状态为 running
        await updateTaskStatus(env.DB, task.taskId, 'running', {
          started_at: Date.now(),
        })

        // 获取对应的处理器
        const processor = registry.get(task.type)

        // 执行任务
        const result = await processor.execute(task)

        // 更新状态为 completed
        await updateTaskStatus(env.DB, task.taskId, 'completed', {
          output_data: JSON.stringify(result),
          progress: 100,
          completed_at: Date.now(),
        })

        // 确认消息消费成功
        message.ack()
      } catch (error) {
        const errorMessage = error instanceof Error ? error.message : 'Unknown error'

        if (task.retryCount < 3) {
          // 标记失败，让 Queue 自动重试
          await updateTaskStatus(env.DB, task.taskId, 'failed', {
            error_message: errorMessage,
            retry_count: task.retryCount + 1,
          })
          message.retry({ delaySeconds: getRetryDelay(task.retryCount) })
        } else {
          // 超过重试次数，进入死信队列
          await updateTaskStatus(env.DB, task.taskId, 'dead_letter', {
            error_message: `最终失败: ${errorMessage}`,
            completed_at: Date.now(),
          })

          // 退还积分
          await refundCredits(env.DB, task.userId, task.creditsCharged, task.taskId)

          message.ack() // 不再重试
        }
      }
    }
  },
}

// 指数退避重试延迟（秒）
function getRetryDelay(retryCount: number): number {
  const base = 30 // 基础延迟 30 秒
  const max = 300 // 最大延迟 5 分钟
  return Math.min(base * Math.pow(2, retryCount), max)
}
```

### 3.6 死信队列处理

死信队列 `ai-tasks-dlq` 用于收集所有彻底失败的任务消息，可用于：

- 事后分析失败原因
- 批量手动重处理
- 监控告警触发

```typescript
// ============================================================
//  死信队列消费者（可选的独立 Worker，或定期 Cron）
//  workers/dlq-handler/src/index.ts
// ============================================================

export default {
  async queue(batch: MessageBatch<TaskMessage>, env: Env): Promise<void> {
    for (const message of batch.messages) {
      const task = message.body

      // 记录到专门的失败日志表
      await env.DB.prepare(
        `
        INSERT INTO task_failures (task_id, user_id, type, error_data, created_at)
        VALUES (?, ?, ?, ?, ?)
      `,
      )
        .bind(task.taskId, task.userId, task.type, JSON.stringify(task), Date.now())
        .run()

      // 发送告警（可接入 Webhook / Slack / 邮件）
      await sendAlert(env, {
        level: 'error',
        title: `任务彻底失败: ${task.type}`,
        taskId: task.taskId,
        userId: task.userId,
      })

      message.ack()
    }
  },
}
```

### 4.1 处理器接口设计

```typescript
/**
 * 任务处理结果
 */
interface TaskResult {
  /** 输出文件 URL 列表 */
  files: Array<{
    url: string
    type: 'video' | 'image' | 'audio'

/**
 * 任务处理器接口
 *
 * 所有 AI 任务处理器必须实现此接口。
 * 处理器负责：调用外部 API → 轮询任务状态 → 下载结果 → 上传到 R2
 */
interface TaskProcessor {
  /** 处理器支持的任务类型 */
  readonly type: string

  /**
   * 执行任务
   * @param task - 队列消息中的任务数据
   * @returns 任务处理结果
   * @throws 任务失败时抛出异常
   */
  execute(task: TaskMessage): Promise<TaskResult>
}
```

### 4.2 处理器基类

```typescript
// ============================================================
//  处理器基类（提供公共能力）
//  workers/task-consumer/src/processors/abstract.ts
// ============================================================

abstract class AbstractProcessor implements TaskProcessor {
  abstract readonly type: string

  constructor(protected env: Env) {}

  abstract execute(task: TaskMessage): Promise<TaskResult>

  // ----------------------------------------------------------
  //  公共方法: 轮询外部任务状态（指数退避）
  // ----------------------------------------------------------
  protected async pollExternalTask<T>(
    checkFn: () => Promise<{ done: boolean; result?: T; progress?: number }>,
    taskId: string,
    timeoutMs: number,
  ): Promise<T> {
    const startTime = Date.now()
    let interval = 3000 // 初始轮询间隔 3s
    const maxInterval = 30000 // 最大轮询间隔 30s
    const backoffFactor = 1.5 // 退避因子

    while (Date.now() - startTime < timeoutMs) {
      const { done, result, progress } = await checkFn()

      // 更新进度
      if (progress !== undefined) {
        await this.updateProgress(taskId, progress)
      }

      if (done) {
        if (!result) throw new Error('任务完成但无结果返回')
        return result
      }

      // 等待后继续轮询
      await sleep(interval)
      interval = Math.min(interval * backoffFactor, maxInterval)
    }

    throw new Error(`轮询超时: ${timeoutMs}ms`)
  }

  // ----------------------------------------------------------
  //  公共方法: 下载外部文件并上传到 R2
  // ----------------------------------------------------------
  protected async downloadToR2(
    sourceUrl: string,
    userId: string,
    taskId: string,
    filename: string,
  ): Promise<string> {
    // 下载外部文件
    const response = await fetch(sourceUrl)
    if (!response.ok) {
      throw new Error(`下载失败: ${response.status} ${response.statusText}`)
    }
    const data = await response.arrayBuffer()

    // 构造 R2 存储路径
    const r2Key = `users/${userId}/tasks/${taskId}/${filename}`

    // 上传到 R2
    await this.env.BUCKET.put(r2Key, data, {
      httpMetadata: {
        contentType: response.headers.get('content-type') || 'application/octet-stream',
      },
    })

    // 返回 R2 公开 URL
    return `${this.env.R2_PUBLIC_URL}/${r2Key}`
  }

  // ----------------------------------------------------------
  //  公共方法: 更新任务进度
  // ----------------------------------------------------------
  protected async updateProgress(taskId: string, progress: number): Promise<void> {
    await this.env.DB.prepare('UPDATE async_tasks SET progress = ? WHERE id = ?')
      .bind(Math.min(progress, 100), taskId)
      .run()
  }
}

function sleep(ms: number): Promise<void> {
  return new Promise((resolve) => setTimeout(resolve, ms))
}
```

### 4.3 VideoGenProcessor

```typescript
// ============================================================
//  视频生成处理器
//  workers/task-consumer/src/processors/video-gen.ts
// ============================================================

/**
 * 支持的视频生成提供商：Kling、Wan（通义万相）
 *
 * 共同特征：异步 API，需要先提交任务，再轮询结果
 *
 * Kling 流程：POST /v1/videos/text2video → GET /v1/videos/{task_id}
 * Wan 流程：POST (阿里云 DashScope) → GET task status
 */
class VideoGenProcessor extends AbstractProcessor {
  readonly type = 'video_gen'

  async execute(task: TaskMessage): Promise<TaskResult> {
    const config = TIMEOUT_CONFIG.video_gen

    // ----------------------------------------------------------
    // Step 1: 提交视频生成任务到外部 API
    // ----------------------------------------------------------
    const externalTaskId = await this.submitTask(task)

    // 记录外部任务 ID
    await this.env.DB.prepare('UPDATE async_tasks SET external_task_id = ? WHERE id = ?')
      .bind(externalTaskId, task.taskId)
      .run()

    // ----------------------------------------------------------
    // Step 2: 轮询外部任务状态
    // ----------------------------------------------------------
    const result = await this.pollExternalTask<{ videoUrl: string }>(
      () => this.checkTaskStatus(task.provider, externalTaskId),
      task.taskId,
      config.pollingTimeout,
    )

    // ----------------------------------------------------------
    // Step 3: 下载视频并上传到 R2
    // ----------------------------------------------------------
    const filename = `output_${Date.now()}.mp4`
    const r2Url = await this.downloadToR2(
      result.videoUrl,
      task.userId,
      task.taskId,
      filename,
    )

    return {
      files: [
        {
          url: r2Url,
          type: 'video',
          filename,
        },
      ],
      metadata: {
        externalTaskId,
        provider: task.provider,
      },
    }
  }

  private async submitTask(task: TaskMessage): Promise<string> {
    const { provider, modelId, input } = task

    switch (provider) {
      case 'kling':
        return this.submitKlingTask(modelId, input)
      case 'wan':
        return this.submitWanTask(modelId, input)
      default:
        throw new Error(`不支持的视频生成提供商: ${provider}`)
    }
  }

  private async submitKlingTask(
    modelId: string,
    input: Record<string, unknown>,
  ): Promise<string> {
    const response = await fetch('https://api.klingai.com/v1/videos/text2video', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        Authorization: `Bearer ${this.env.KLING_API_KEY}`,
      },
      body: JSON.stringify({
```

```typescript
/**
 * 支持的图片生成提供商：QWEN（通义）、Z-Image

class ImageGenProcessor extends AbstractProcessor {
  readonly type = 'image_gen'

  async execute(task: TaskMessage): Promise<TaskResult> {
    const config = TIMEOUT_CONFIG.image_gen
    const { provider, modelId, input } = task

    // 提交任务
    const externalTaskId = await this.submitImageTask(provider, modelId, input)

    await this.env.DB.prepare('UPDATE async_tasks SET external_task_id = ? WHERE id = ?')
      .bind(externalTaskId, task.taskId)
      .run()

    // 轮询结果
    const result = await this.pollExternalTask<{ imageUrls: string[] }>(
      () => this.checkImageStatus(provider, externalTaskId),
      task.taskId,
      config.pollingTimeout,
    )

    // 下载所有图片到 R2
    const files = await Promise.all(
      result.imageUrls.map(async (url, index) => {
        const filename = `output_${Date.now()}_${index}.png`
        const r2Url = await this.downloadToR2(url, task.userId, task.taskId, filename)
        return { url: r2Url, type: 'image' as const, filename }
      }),
    )

    return { files }
  }

    modelId: string,
```

## 六、客户端状态推送

### 6.1 方案对比

| 维度                      | 方案一：轮询                   | 方案二：SSE                                        |
| ------------------------- | ------------------------------ | -------------------------------------------------- |
| 实现复杂度                | 低                             | 中                                                 |
| 实时性                    | 3-5s 延迟                      | 近实时                                             |
| 服务端资源                | 每次请求独立                   | 需保持长连接                                       |
| Cloudflare Workers 兼容性 | 完全兼容                       | 有限制（最长 30s 连接，需 Durable Objects 持久化） |
| 客户端实现                | TanStack Query refetchInterval | EventSource API                                    |
| 移动端兼容                | 完全兼容                       | 部分浏览器有问题                                   |

### 6.2 推荐方案：轮询

**选择理由**：

- Cloudflare Workers 不原生支持长连接 SSE（超过 30s 会断开）
- SSE 需要引入 Durable Objects，增加架构复杂度和成本
- AI 任务本身就是分钟级别的，3-5s 轮询延迟完全可接受
- TanStack Query 的 `refetchInterval` 开箱即用

### 6.3 前端轮询实现

```typescript
// ============================================================
//  hooks/api/useTaskPolling.ts
//  使用 TanStack Query 实现任务状态轮询
// ============================================================

import { useQuery } from '@tanstack/react-query'

interface TaskStatus {
  id: string
  status: 'pending' | 'queued' | 'running' | 'completed' | 'failed' | 'dead_letter'
  progress: number
  outputData?: TaskOutput
  errorMessage?: string
  createdAt: number
  startedAt?: number
  completedAt?: number
}

/**
 * 轮询单个任务状态
 *
 * - 任务处于活跃状态时：每 3 秒轮询
 * - 任务已完成/失败时：停止轮询
 */
export function useTaskStatus(taskId: string | null) {
  return useQuery({
    queryKey: ['task', taskId],
    queryFn: async (): Promise<TaskStatus> => {
      const response = await fetch(`/api/tasks/${taskId}`)
      if (!response.ok) throw new Error('获取任务状态失败')
      return response.json()
    },
    enabled: !!taskId,
    refetchInterval: (query) => {
      const data = query.state.data
      if (!data) return 3000

      // 活跃状态：3 秒轮询
      const activeStatuses = ['pending', 'queued', 'running']
      if (activeStatuses.includes(data.status)) return 3000

      // 终态：停止轮询
      return false
    },
    staleTime: 2000, // 2 秒内认为数据新鲜
    gcTime: 5 * 60_000, // 垃圾回收时间 5 分钟
  })
}

/**
 * 轮询用户所有活跃任务
 *
 * 用于全局任务面板 / 通知栏
 */
export function useActiveTasks() {
  return useQuery({
    queryKey: ['tasks', 'active'],
    queryFn: async (): Promise<TaskStatus[]> => {
      const response = await fetch('/api/tasks?status=active')
      if (!response.ok) throw new Error('获取任务列表失败')
      return response.json()
    },
    refetchInterval: 5000, // 5 秒轮询
    staleTime: 3000,
  })
}
```

```typescript
// ============================================================
//  lib/concurrency.ts
//  并发控制与用户套餐权限
// ============================================================

const PLAN_LIMITS: Record<string, number> = {
  free: 1,
  standard: 2,
  pro: 4,
  ultimate: 8,
}

/**
 * 获取用户最大并发任务数
 */
async function getUserConcurrentLimit(db: D1Database, userId: string): Promise<number> {
  const sub = await db
    .prepare('SELECT plan FROM subscriptions WHERE user_id = ? AND status = ?')
    .bind(userId, 'active')
    .first<{ plan: string }>()

  const plan = sub?.plan ?? 'free'
  return PLAN_LIMITS[plan] ?? 1
}

/**
 * 检查用户是否可以提交新任务
 */
async function canSubmitTask(
  db: D1Database,
  userId: string,
): Promise<{
  allowed: boolean
  reason?: string
  currentCount: number
  maxCount: number
}> {
  const maxConcurrent = await getUserConcurrentLimit(db, userId)

  const result = await db
    .prepare(
      `
      SELECT COUNT(*) as count FROM async_tasks
      WHERE user_id = ? AND status IN ('pending', 'queued', 'running')
    `,
    )
    .bind(userId)
    .first<{ count: number }>()

  const currentCount = result?.count ?? 0

  if (currentCount >= maxConcurrent) {
    return {
      allowed: false,
      reason: `并发任务数已达上限 (${currentCount}/${maxConcurrent})，请等待当前任务完成或升级套餐`,
      currentCount,
      maxCount: maxConcurrent,
    }
  }

  return { allowed: true, currentCount, maxCount: maxConcurrent }
}
```

### 7.3 队列优先级

### 8.4 过期清理（Lifecycle Rules）

通过 Cloudflare R2 Object Lifecycle Rules 自动清理过期文件：

| 用户套餐 | 文件保留时间 | 存储配额 |
| -------- | ------------ | -------- |
| Free     | 30 天        | 1 GB     |
| Standard | 90 天        | 10 GB    |
| Pro      | 180 天       | 50 GB    |
| Ultimate | 365 天       | 200 GB   |

```typescript
// ============================================================
//  过期清理 Cron Job (Scheduled Worker)
//  每天凌晨运行，清理过期文件
// ============================================================

export default {
  async scheduled(event: ScheduledEvent, env: Env): Promise<void> {
    // 查询已过期的任务
    const expiredTasks = await env.DB.prepare(
      `
      SELECT t.id, t.user_id, t.output_data
      FROM async_tasks t
      JOIN subscriptions s ON t.user_id = s.user_id
      WHERE t.status = 'completed'
        AND t.completed_at < ?
        AND (
          (s.plan = 'free'     AND t.completed_at < ? - 30 * 86400000) OR
          (s.plan = 'standard' AND t.completed_at < ? - 90 * 86400000) OR
          (s.plan = 'pro'      AND t.completed_at < ? - 180 * 86400000) OR
          (s.plan = 'ultimate' AND t.completed_at < ? - 365 * 86400000)
        )
      LIMIT 100
    `,
    )
      .bind(Date.now(), Date.now(), Date.now(), Date.now(), Date.now())
      .all()

    for (const task of expiredTasks.results) {
      // 删除 R2 文件
      const prefix = `users/${task.user_id}/tasks/${task.id}/`
      const objects = await env.BUCKET.list({ prefix })
      for (const obj of objects.objects) {
        await env.BUCKET.delete(obj.key)
      }

      // 标记任务为已清理
      await env.DB.prepare('UPDATE async_tasks SET output_data = NULL WHERE id = ?')
        .bind(task.id)
        .run()
    }
  },
}
```

```typescript
// ============================================================
//  GET /api/tasks - 查询用户任务列表
//  app/api/tasks/route.ts (GET handler)
// ============================================================

export async function GET(request: Request) {
  const { userId } = auth()
  if (!userId) return Response.json({ error: 'Unauthorized' }, { status: 401 })

  const url = new URL(request.url)
  const status = url.searchParams.get('status') ?? 'all'
  const type = url.searchParams.get('type')
  const page = Math.max(1, parseInt(url.searchParams.get('page') ?? '1'))
  const pageSize = Math.min(
    50,
    Math.max(1, parseInt(url.searchParams.get('pageSize') ?? '20')),
  )
  const offset = (page - 1) * pageSize

  // 构建查询条件
  let whereClause = 'WHERE user_id = ?'
  const bindings: unknown[] = [userId]

  if (status === 'active') {
    whereClause += " AND status IN ('pending', 'queued', 'running')"
  } else if (status === 'completed') {
    whereClause += " AND status = 'completed'"
  } else if (status === 'failed') {
    whereClause += " AND status IN ('failed', 'dead_letter')"
  }

  if (type) {
    whereClause += ' AND type = ?'
    bindings.push(type)
  }

  // 查询总数
  const countResult = await db
    .prepare(`SELECT COUNT(*) as total FROM async_tasks ${whereClause}`)
    .bind(...bindings)
    .first<{ total: number }>()
  const total = countResult?.total ?? 0

  // 查询数据
  const tasks = await db
    .prepare(
      `
      SELECT * FROM async_tasks ${whereClause}
      ORDER BY created_at DESC
      LIMIT ? OFFSET ?
    `,
    )
    .bind(...bindings, pageSize, offset)
    .all()

  return Response.json({
    tasks: tasks.results.map(formatTaskResponse),
    total,
    page,
    pageSize,
    hasMore: offset + pageSize < total,
  } satisfies TaskListResponse)
}

// ============================================================
//  DELETE /api/tasks/:id - 取消任务
//  app/api/tasks/[id]/route.ts
// ============================================================

export async function DELETE(request: Request, { params }: { params: { id: string } }) {
  const { userId } = auth()
  if (!userId) return Response.json({ error: 'Unauthorized' }, { status: 401 })

  const task = await db
    .prepare('SELECT * FROM async_tasks WHERE id = ? AND user_id = ?')
    .bind(params.id, userId)
    .first()

  if (!task) {
    return Response.json({ error: 'Task not found' }, { status: 404 })
  }

  // 只有 pending 和 queued 状态可以取消
  if (!['pending', 'queued'].includes(task.status as string)) {
    return Response.json(
      {
        success: false,
        message: `无法取消 ${task.status} 状态的任务`,
      },
      { status: 400 },
    )
  }

  // 退还积分
  const credits = task.credits_charged as number
  await refundCredits(db, userId, credits, task.id as string)

  // 更新状态
  await db
    .prepare(
      "UPDATE async_tasks SET status = 'failed', error_message = '用户取消', completed_at = ? WHERE id = ?",
    )
    .bind(Date.now(), params.id)
    .run()

  return Response.json({
    success: true,
    message: '任务已取消',
    creditsRefunded: credits,
  } satisfies CancelTaskResponse)
}
```

## 十、错误处理与重试

### 10.1 重试策略

```typescript
// ============================================================
//  重试策略配置
// ============================================================

const RETRY_CONFIG = {
  maxRetries: 3, // 最多重试 3 次

  // 指数退避延迟（秒）
  // 第 1 次重试：30s，第 2 次：60s，第 3 次：120s
  getDelay(retryCount: number): number {
    return Math.min(30 * Math.pow(2, retryCount), 300)
  },

  // 可重试的错误类型
  isRetryable(error: unknown): boolean {
    if (error instanceof Error) {
      const message = error.message.toLowerCase()

      // 不可重试：客户端错误
      if (message.includes('invalid') || message.includes('unauthorized')) {
        return false
      }
      // 不可重试：内容违规
      if (message.includes('content_policy') || message.includes('moderation')) {
        return false
      }

      // 可重试：超时、网络错误、服务端错误
      return true
    }
    return true
  },
}
```

### 10.2 超时处理

```typescript
// ============================================================
//  超时检测 Scheduled Worker
//  每 5 分钟运行，检测并标记超时任务
// ============================================================

export default {
  async scheduled(event: ScheduledEvent, env: Env): Promise<void> {
    const now = Date.now()

    // 查找 running 但超时的任务
    const timeoutTasks = await env.DB.prepare(
      `
      SELECT id, type, user_id, credits_charged, started_at
      FROM async_tasks
      WHERE status = 'running' AND started_at IS NOT NULL
    `,
    ).all()

    for (const task of timeoutTasks.results) {
      const taskType = task.type as string
      const startedAt = task.started_at as number
      const timeout = TIMEOUT_CONFIG[taskType]?.executionTimeout ?? 15 * 60 * 1000

      if (now - startedAt > timeout) {
        // 标记为失败
        await env.DB.prepare(
          `
          UPDATE async_tasks
          SET status = 'failed',
              error_message = '任务执行超时',
              completed_at = ?
          WHERE id = ?
        `,
        )
          .bind(now, task.id)
          .run()

        // 退还积分
        await refundCredits(
          env.DB,
          task.user_id as string,
          task.credits_charged as number,
          task.id as string,
        )
      }
    }
  },
}
```

### 10.3 积分退还流程

```typescript
// ============================================================
//  积分退还
//  lib/credits.ts
// ============================================================

/**
 * 退还积分
 *
 * 触发条件：
 * 1. 任务进入 dead_letter 状态（彻底失败）
 * 2. 任务超时
 * 3. 用户主动取消（pending/queued 状态）
 *
 * 注意：running 状态的任务不可取消（已消耗外部 API 资源）
 */
async function refundCredits(
  db: D1Database,
  userId: string,
  amount: number,
  taskId: string,
): Promise<void> {
  if (amount <= 0) return

  // 更新余额
  await db
    .prepare(
      `
    UPDATE credit_balances
    SET balance = balance + ?,
        total_earned = total_earned + ?,
        updated_at = ?
    WHERE user_id = ?
  `,
    )
    .bind(amount, amount, Date.now(), userId)
    .run()

  // 记录退款交易
  await db
    .prepare(
      `
    INSERT INTO credit_transactions (id, user_id, type, amount, balance_after, source, reference_id, description, created_at)
    VALUES (?, ?, 'refund', ?, (SELECT balance FROM credit_balances WHERE user_id = ?), 'refund', ?, ?, ?)
  `,
    )
    .bind(
      nanoid(),
      userId,
      amount,
      userId,
      taskId,
      `任务失败退还积分 (任务 ${taskId})`,
      Date.now(),
    )
    .run()
}
```

## 十一、监控与告警

### 11.1 核心指标

| 指标         | 计算方式                              | 健康阈值 | 告警阈值 |
| ------------ | ------------------------------------- | -------- | -------- |
| 任务成功率   | completed / (completed + dead_letter) | > 95%    | < 90%    |
| 平均处理时间 | avg(completed_at - created_at)        | < 5min   | > 15min  |
| 队列积压     | count(status IN pending, queued)      | < 50     | > 200    |
| 重试率       | sum(retry_count > 0) / total          | < 10%    | > 25%    |
| 死信率       | count(dead_letter) / total            | < 2%     | > 5%     |

### 11.2 监控查询

```sql
-- ============================================================
--  监控仪表盘 SQL 查询
-- ============================================================

-- 过去 24 小时任务成功率
SELECT
  ROUND(
    COUNT(CASE WHEN status = 'completed' THEN 1 END) * 100.0 /
    NULLIF(COUNT(CASE WHEN status IN ('completed', 'dead_letter') THEN 1 END), 0),
    2
  ) AS success_rate
FROM async_tasks
WHERE created_at > ? - 86400000;

-- 过去 24 小时平均处理时间（毫秒）
SELECT
  type,
  AVG(completed_at - created_at) AS avg_duration_ms,
  MIN(completed_at - created_at) AS min_duration_ms,
  MAX(completed_at - created_at) AS max_duration_ms,
  COUNT(*) AS count
FROM async_tasks
WHERE status = 'completed'
  AND created_at > ? - 86400000
GROUP BY type;

-- 当前队列积压
SELECT
  status,
  COUNT(*) AS count
FROM async_tasks
WHERE status IN ('pending', 'queued', 'running')
GROUP BY status;

-- 过去 1 小时按提供商分布的失败率
SELECT
  provider,
  COUNT(*) AS total,
  COUNT(CASE WHEN status = 'dead_letter' THEN 1 END) AS failures,
  ROUND(
    COUNT(CASE WHEN status = 'dead_letter' THEN 1 END) * 100.0 / COUNT(*),
    2
  ) AS failure_rate
FROM async_tasks
WHERE created_at > ? - 3600000
GROUP BY provider;
```

### 11.3 告警实现

```typescript
// ============================================================
//  告警检查 Scheduled Worker
//  每 10 分钟运行
// ============================================================

export default {
  async scheduled(event: ScheduledEvent, env: Env): Promise<void> {
    const now = Date.now()
    const oneHourAgo = now - 3600000

    // 检查 1: 队列积压
    const backlog = await env.DB.prepare(
      `
      SELECT COUNT(*) as count FROM async_tasks
      WHERE status IN ('pending', 'queued')
    `,
    ).first<{ count: number }>()

    if (backlog && backlog.count > 200) {
      await sendAlert(env, {
        level: 'warning',
        title: '队列积压告警',
        message: `当前积压 ${backlog.count} 个任务`,
      })
    }

    // 检查 2: 失败率过高
    const stats = await env.DB.prepare(
      `
      SELECT
        COUNT(*) as total,
        COUNT(CASE WHEN status = 'dead_letter' THEN 1 END) as failures
      FROM async_tasks
      WHERE created_at > ?
    `,
    )
      .bind(oneHourAgo)
      .first<{ total: number; failures: number }>()

    if (stats && stats.total > 10) {
      const failureRate = stats.failures / stats.total
      if (failureRate > 0.05) {
        await sendAlert(env, {
          level: 'critical',
          title: '任务失败率过高',
          message: `过去 1 小时失败率: ${(failureRate * 100).toFixed(1)}% (${stats.failures}/${stats.total})`,
        })
      }
    }
  },
}

/**
 * 发送告警通知
 * 可接入 Slack / Discord / 邮件 / 企业微信
 */
async function sendAlert(
  env: Env,
  alert: {
    level: string
    title: string
    message?: string
    taskId?: string
    userId?: string
  },
): Promise<void> {
  // Slack Webhook 示例
  if (env.SLACK_WEBHOOK_URL) {
    await fetch(env.SLACK_WEBHOOK_URL, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        text: `[${alert.level.toUpperCase()}] ${alert.title}\n${alert.message ?? ''}`,
      }),
    })
  }
}
```
