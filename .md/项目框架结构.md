## 二、渲染策略

### 2.1 页面渲染模式分配

| 页面         | 路由                   | 渲染模式 | 理由                                   |
| ------------ | ---------------------- | -------- | -------------------------------------- |
| Landing Page | `/`                    | **SSG**  | 纯静态内容，最佳 SEO + 性能            |
| 定价页面     | `/pricing`             | **SSG**  | 静态内容，套餐信息通过 revalidate 更新 |
| 服务条款     | `/terms`               | **SSG**  | 纯静态                                 |
| 隐私政策     | `/privacy`             | **SSG**  | 纯静态                                 |
| 登录/注册    | `/sign-in`, `/sign-up` | **CSR**  | Clerk 组件客户端渲染                   |
| 作品广场     | `/explore`             | **SSR**  | 动态内容需要 SEO，搜索引擎可索引       |
| 作品详情     | `/explore/:id`         | **SSR**  | 每个作品需要独立的 OG 标签             |
| 创作空间     | `/workspace`           | **SSR**  | 用户私有数据，服务端获取               |
| 画布编辑器   | `/workspace/:id`       | **CSR**  | ReactFlow 纯客户端，重交互             |
| 工作流分享   | `/workflows`           | **SSR**  | 分类 + 搜索 + 公开工作流卡片           |
| 视频分析     | `/video-analysis`      | **CSR**  | 上传 + AI 模型 + 历史（框架已搭建）     |
| 元素库       | `/elements`            | **SSR**  | 占位页面                               |

> **个人中心 / 账单**：通过 AppSidebar 底部 ProfileModal 弹窗访问（720x520，6 Tab），非独立路由页面

### 2.2 渲染模式说明

```
SSG (Static Site Generation)
├── 构建时生成 HTML
├── CDN 分发，性能最佳
├── 适用于不频繁变化的内容
└── 通过 revalidate 定期更新

SSR (Server-Side Rendering)
├── 每次请求动态生成 HTML
├── SEO 友好（搜索引擎爬到完整内容）
├── 适用于需要用户数据或频繁变化的页面
└── 运行在 Cloudflare Workers 边缘节点

CSR (Client-Side Rendering)
├── 浏览器端渲染
├── 适用于重交互页面（画布编辑器）
├── 使用 'use client' 指令
└── 不需要 SEO 的页面
```

### 4.1 Next.js 中间件

```typescript
// middleware.ts
import { clerkMiddleware, createRouteMatcher } from '@clerk/nextjs/server'
import createMiddleware from 'next-intl/middleware'

const intlMiddleware = createMiddleware({
  locales: ['en', 'zh'],
  defaultLocale: 'en',
})

const isProtectedRoute = createRouteMatcher([
  '/:locale/(app)(.*)',
  '/:locale/workspace(.*)',
])

export default clerkMiddleware(async (auth, req) => {
  // i18n 路由处理
  const response = intlMiddleware(req)

  // 受保护路由检查
  if (isProtectedRoute(req)) {
    await auth.protect()
  }

  return response
})

export const config = {
  matcher: ['/((?!api|_next|.*\\..*).*)'],
}
```
