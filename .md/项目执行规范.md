## 一、开发流程规范

### 1.1 任务执行流程

```
1. 查看「项目规划清单.md」确认当前任务
2. 阅读任务相关的设计文档
3. 按照「项目框架结构.md」创建/修改文件
4. 编写代码并自测
5. 更新「项目规划清单.md」中的任务状态
6. 提交代码（如使用 Git）
```

### 1.2 任务状态标记

| 标记 | 含义 |
|-----|------|
| `[ ]` | 未开始 |
| `[~]` | 进行中 |
| `[x]` | 已完成 |
| `[!]` | 遇到阻塞 |
| `[-]` | 已取消 |

### 1.3 开发优先级

1. **先完成依赖项**：参考「项目规划清单」中的依赖关系图
2. **先做核心功能**：MVP 任务优先于 P1、P2
3. **先通后优**：先让功能可用，再优化体验

### 2.3 组件编写规范

```typescript
// 1. 导入顺序：React → 第三方库 → 本地模块 → 类型 → 样式
import { useState, useCallback } from 'react'
import { Handle, Position } from 'reactflow'
import { useFlowStore } from '@/stores'
import type { NodeProps } from '@/types'
import './TextInputNode.css'

// 2. 类型定义
interface TextInputNodeData {
  value: string
}

// 3. 组件定义（使用函数声明）
export function TextInputNode({ id, data, selected }: NodeProps<TextInputNodeData>) {
  // 4. Hooks 在最顶部
  const updateNode = useFlowStore((state) => state.updateNode)
  const [localValue, setLocalValue] = useState(data.value)

  // 5. 事件处理函数
  const handleChange = useCallback((e: React.ChangeEvent<HTMLTextAreaElement>) => {
    setLocalValue(e.target.value)
    updateNode(id, { value: e.target.value })
  }, [id, updateNode])

  // 6. 渲染
  return (
    <div className={`text-input-node ${selected ? 'selected' : ''}`}>
      <div className="node-header">文本输入</div>
      <textarea value={localValue} onChange={handleChange} />
      <Handle type="source" position={Position.Right} id="output" />
    </div>
  )
}
```

### 2.4 状态管理规范

```typescript
// stores/useFlowStore.ts
import { create } from 'zustand'
import { persist } from 'zustand/middleware'
import type { Node, Edge, Viewport } from '@/types'

interface FlowState {
  // 状态
  nodes: Node[]
  edges: Edge[]
  viewport: Viewport

  // 操作
  addNode: (node: Node) => void
  updateNode: (id: string, data: Partial<Node['data']>) => void
  removeNode: (id: string) => void
  // ...
}

export const useFlowStore = create<FlowState>()(
  persist(
    (set, get) => ({
      nodes: [],
      edges: [],
      viewport: { x: 0, y: 0, zoom: 1 },

      addNode: (node) => set((state) => ({
        nodes: [...state.nodes, node]
      })),

      updateNode: (id, data) => set((state) => ({
        nodes: state.nodes.map((node) =>
          node.id === id
            ? { ...node, data: { ...node.data, ...data } }
            : node
        )
      })),

      removeNode: (id) => set((state) => ({
        nodes: state.nodes.filter((node) => node.id !== id),
        edges: state.edges.filter(
          (edge) => edge.source !== id && edge.target !== id
        )
      })),
    }),
    {
      name: 'nano-banana-canvas-storage',
    }
  )
)
```

### 2.6 错误处理规范

```typescript
// 1. AI 服务调用错误处理
try {
  const result = await openRouterService.chat(params)
  return result
} catch (error) {
  if (error instanceof APIError) {
    // API 错误，显示具体信息
    toast.error(`API 错误: ${error.message}`)
  } else if (error instanceof NetworkError) {
    // 网络错误，提示重试
    toast.error('网络连接失败，请检查网络后重试')
  } else {
    // 未知错误
    toast.error('发生未知错误')
    console.error(error)
  }
  throw error
}

// 2. 节点执行错误处理
async function executeNode(node: Node, inputs: Record<string, any>) {
  try {
    updateNodeStatus(node.id, 'running')
    const outputs = await nodeExecutors[node.type](node, inputs)
    updateNodeStatus(node.id, 'success')
    updateNodeOutputs(node.id, outputs)
    return outputs
  } catch (error) {
    updateNodeStatus(node.id, 'error')
    updateNodeError(node.id, error.message)
    throw error
  }
}
```

## 三、UI/UX 规范

### 3.1 设计原则

1. **简洁优先**：界面保持干净，避免视觉噪音
2. **即时反馈**：用户操作后立即给予视觉反馈
3. **容错设计**：支持撤销，危险操作需确认
4. **一致性**：相同功能使用相同的交互模式

test.describe('Canvas', () => {

## 五、Git 规范

### 5.1 分支命名

| 类型 | 格式 | 示例 |
|-----|------|------|
| 功能开发 | `feat/功能名` | `feat/llm-node` |
| Bug 修复 | `fix/问题描述` | `fix/edge-delete` |
| 重构 | `refactor/模块名` | `refactor/executor` |
| 文档 | `docs/内容` | `docs/readme` |

### 5.2 Commit 规范

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type 类型**：
- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 代码格式（不影响功能）
- `refactor`: 重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具变动

**示例**：
```
feat(nodes): 添加 LLM 生成节点

- 实现节点 UI 和配置面板
- 集成 OpenRouter API
- 支持流式输出

Closes #12
```

### 5.3 版本号规范

遵循 Semantic Versioning：`MAJOR.MINOR.PATCH`

| 类型 | 变更条件 |
|-----|---------|
| MAJOR | 不兼容的 API 变更 |
| MINOR | 向后兼容的新功能 |
| PATCH | 向后兼容的 Bug 修复 |

## 七、问题处理流程

### 7.1 遇到阻塞时

1. 记录问题详情
2. 在任务清单中标记 `[!]`
3. 分析问题原因
4. 尝试替代方案
5. 如无法解决，暂时跳过，继续其他任务

1. 评估变更影响范围
2. 更新相关文档
3. 调整任务清单
4. 重新评估优先级
