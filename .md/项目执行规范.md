## 一、开发流程规范

### 1.1 任务执行流程

```
1. 查看「项目规划清单.md」确认当前任务
2. 阅读任务相关的设计文档
3. 按照「项目框架结构.md」创建/修改文件
4. 编写代码并自测
5. 更新「项目规划清单.md」中的任务状态
```

### 1.2 任务状态标记

| 标记  | 含义     |
| ----- | -------- |
| `[ ]` | 未开始   |
| `[~]` | 进行中   |
| `[x]` | 已完成   |
| `[!]` | 遇到阻塞 |

## 二、代码规范

### 2.1 文件命名

| 类型           | 规则                    | 示例                                    |
| -------------- | ----------------------- | --------------------------------------- |
| Next.js 页面   | `page.tsx` (App Router) | `app/[locale]/(app)/workspace/page.tsx` |

- 页面放在 `app/[locale]/` 下对应路由组
- 组件放在 `components/` 下按功能分子目录

- 工具函数放在 `lib/` 目录
- AI 服务放在 `services/ai/` 目录
- Zustand Store 放在 `stores/` 目录

### 2.3 Next.js App Router 规范

#### Server Component vs Client Component

```typescript
// 默认是 Server Component（SSR/SSG），可以直接访问 D1/R2
// app/[locale]/(app)/workspace/page.tsx
import { auth } from '@clerk/nextjs/server'

export default async function WorkspacePage() {
  const { userId } = await auth()
  // 直接查询数据库
  const workflows = await db.query('SELECT * FROM workflows WHERE user_id = ?', [userId])

  return <WorkflowList workflows={workflows} />
}
```

```typescript
// 需要交互（useState, onClick, 浏览器 API）才用 'use client'
// components/canvas/canvas.tsx
'use client'

import { useState, useCallback } from 'react'
import ReactFlow from 'reactflow'

export function Canvas() {
  const [nodes, setNodes] = useState([])
  // ...
}
```

**原则**：能用 Server Component 就用 Server Component。只有需要浏览器交互的组件才标记 `'use client'`。

#### 数据获取

```typescript
// Server Component 直接 fetch（推荐）
export default async function ExplorePage() {
  const workflows = await fetch('/api/explore').then(r => r.json())
  return <WorkflowGrid workflows={workflows} />
}

// Client Component 用 TanStack Query
'use client'
import { useQuery } from '@tanstack/react-query'

export function WorkflowList() {
  const { data, isLoading } = useQuery({
    queryKey: ['workflows'],
    queryFn: () => fetch('/api/workflows').then(r => r.json()),
  })
  // ...
}
```

### 2.5 API Route Handler 规范

```typescript
// app/api/workflows/route.ts
import { auth } from '@clerk/nextjs/server'
import { NextRequest, NextResponse } from 'next/server'

export async function GET(request: NextRequest) {
  const { userId } = await auth()
  if (!userId) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
  }

  // 访问 D1（通过 Cloudflare bindings）
  const { results } = await process.env.DB.prepare(
    'SELECT * FROM workflows WHERE user_id = ? ORDER BY updated_at DESC',
  )
    .bind(userId)
    .all()

  return NextResponse.json({ data: results })
}

export async function POST(request: NextRequest) {
  const { userId } = await auth()
  if (!userId) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
  }

  const body = await request.json()
  // Zod 验证
  const validated = workflowCreateSchema.parse(body)

  // 创建工作流
  const id = generateId()
  await process.env.DB.prepare(
    'INSERT INTO workflows (id, user_id, name, data, created_at, updated_at) VALUES (?, ?, ?, ?, ?, ?)',
  )
    .bind(
      id,
      userId,
      validated.name,
      JSON.stringify(validated.data),
      Date.now(),
      Date.now(),
    )
    .run()

  return NextResponse.json({ data: { id } }, { status: 201 })
}
```

### 2.6 状态管理规范

```typescript
// stores/use-flow-store.ts
import { create } from 'zustand'
import type { Node, Edge, Viewport } from '@/types'

interface FlowState {
  nodes: Node[]
  edges: Edge[]
  viewport: Viewport

  addNode: (node: Node) => void
  updateNode: (id: string, data: Partial<Node['data']>) => void
  removeNode: (id: string) => void
}

export const useFlowStore = create<FlowState>()((set) => ({
  nodes: [],
  edges: [],
  viewport: { x: 0, y: 0, zoom: 1 },

  addNode: (node) =>
    set((state) => ({
      nodes: [...state.nodes, node],
    })),

  updateNode: (id, data) =>
    set((state) => ({
      nodes: state.nodes.map((node) =>
        node.id === id ? { ...node, data: { ...node.data, ...data } } : node,
      ),
    })),

  removeNode: (id) =>
    set((state) => ({
      nodes: state.nodes.filter((n) => n.id !== id),
      edges: state.edges.filter((e) => e.source !== id && e.target !== id),
    })),
}))
```

### 2.7 错误处理规范

```typescript
// AI 服务调用
try {
  const result = await aiService.execute(params)
  return result
} catch (error) {
  if (error instanceof InsufficientCreditsError) {
    toast.error(t('errors.insufficientCredits'))
  } else if (error instanceof APIError) {
    toast.error(t('errors.apiError', { message: error.message }))
  } else if (error instanceof NetworkError) {
    toast.error(t('errors.networkError'))
  } else {
    toast.error(t('errors.unknown'))
    console.error(error)
  }
  throw error
}
```

## 三、UI/UX 规范

### 3.1 设计原则

1. **简洁优先**：界面干净，避免视觉噪音
2. **即时反馈**：操作后立即给予视觉反馈
3. **容错设计**：支持撤销，危险操作需确认
4. **一致性**：相同功能相同交互
5. **响应式**：Landing Page 全响应式，画布桌面端专属

### 3.2 颜色规范

**基础主题**：Lux 主题（tweakcn），通过 `pnpm dlx shadcn@latest add` 安装。
**品牌色覆盖**：Indigo-500 (#6366F1) 作为主色调。
**Landing 配色**：深色科技风，详见《Landing Page 视觉策略.md》。

```css
:root {
  /* 品牌色（覆盖 Lux 主题默认值） */
  --color-primary: #6366f1; /* Indigo-500 */
  --color-primary-hover: #4f46e5; /* Indigo-600 */
  --color-success: #22c55e;
  --color-warning: #f59e0b;
  --color-error: #ef4444;
}
```

**色彩禁忌**（全项目强制）：

- 禁止使用荧光色 / Neon 色
- 禁止大面积高饱和色块
- 禁止彩虹渐变
