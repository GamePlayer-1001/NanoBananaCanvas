## 一、架构决策记录

| 编号    | 决策项      | 选择                                  | 理由                                                              |
| ------- | ----------- | ------------------------------------- | ----------------------------------------------------------------- |
| ADR-001 | AI 执行模式 | 双模式并存（积分制 + 用户自带 Key）   | 积分制对标 martini.art 商业模式，自带 Key 降低用户门槛            |
| ADR-002 | 前端框架    | Next.js (App Router)                  | SSG/SSR/CSR 混合渲染，SEO 友好，Clerk 官方 SDK，部署到 Cloudflare |
| ADR-003 | 国际化      | P1 阶段中英双语 (next-intl)           | 从架构层面内建 i18n，避免后期大规模重构                           |
| ADR-004 | 支付系统    | Stripe                                | 国际主流，支持订阅 + 一次性支付，martini.art 同款                 |
| ADR-005 | 后端        | Cloudflare Workers + D1 + R2          | Serverless 全家桶，成本可控，异步任务用 D1-as-Queue (Method C)    |
| ADR-006 | 认证        | Clerk (Next.js SDK)                   | 开箱即用，支持 Webhook 同步                                       |

## 三、原致命缺失 — 已全部完成

> **v9.1 审计确认：以下 4 项致命缺失已全部实现为生产级代码。**

### 3.1 积分计费系统 ✅ 已完成

**实现方案**：freeze/confirm/refund 三阶段事务 + D1 batch 原子性
- credit_balances 双池（monthly/permanent），消耗优先月度
- credit_transactions 完整审计链
- model_pricing 18 模型定价
- Stripe 订阅 (weekly/monthly/yearly) + Webhook 幂等处理
- Worker Cron 自动解冻超时积分 (FREEZE_TTL 5min)

### 3.2 服务端 AI 执行架构 ✅ 已完成

**实现方案**：双模式并存 (积分/自带 Key)
- POST /api/ai/execute 同步执行 + POST /api/ai/stream SSE 流式
- AIProvider 抽象层 (OpenRouter/DeepSeek/Gemini/Kling 四 Provider)
- 基类继承 BaseOpenAICompatible，子类仅需 endpoint override
- 用户 Key AES-256-GCM 加密存储

### 3.3 异步任务队列 ✅ 已完成

**实现方案**：D1-as-Queue + 客户端驱动懒评估轮询 (Method C)
- submitTask → checkTask → cancelTask 全生命周期
- Provider 处理器层 (video-gen/image-gen/audio-gen)
- 并发控制 (Free=1/Pro=4)
- 失败即终态 + 即时退款 (fail-fast)
- Worker Cron 超时扫描 + 文件清理

### 3.4 定价页面与套餐设计 ✅ 已完成

**实现方案**：Free + Pro 双档 (简化自 4 档)
- PricingContent 周/月/年切换 + Stripe Checkout
- 功能权限矩阵 (模型访问/并发数/存储配额)
- 首次注册 Free 200 积分

## 五、框架迁移影响（Vite → Next.js）

### 5.1 项目结构重构

| 变化     | 旧 (Vite)                 | 新 (Next.js)                              |
| -------- | ------------------------- | ----------------------------------------- |
| 路由     | React Router (手动配置)   | App Router (文件系统路由)                 |
| 页面位置 | `src/pages/` 组件         | `app/` 目录                               |
| API 路由 | 独立 Worker               | Next.js Route Handlers + Worker (AI代理)  |
| 渲染     | 纯 CSR                    | SSG (Landing) + SSR (广场) + CSR (画布)   |
| 入口     | `main.tsx` + `index.html` | `app/layout.tsx`                          |
| 部署     | Cloudflare Pages (静态)   | Cloudflare Pages (@opennextjs/cloudflare) |
| Clerk    | `@clerk/clerk-react`      | `@clerk/nextjs` (功能更全)                |
| 中间件   | Worker 内 auth.ts         | `middleware.ts` (边缘运行)                |
| i18n     | 无                        | `next-intl`                               |

### 5.2 保留不变的部分

- ReactFlow 画布引擎
- Zustand 状态管理
- Tailwind CSS
- shadcn/ui 组件
- D1 / R2 / Cloudflare 基础设施
- 多模态类型系统

### 5.3 Worker 职责变化

**旧设计**：Worker = 全部后端 API
**新设计**：Worker 仅负责以下职责（Next.js Route Handlers 处理其余）

| 组件                   | 职责                                        |
| ---------------------- | ------------------------------------------- |
| Next.js Route Handlers | 所有 API：CRUD、积分、AI 代理、支付、异步任务、Webhook |
| Cloudflare D1          | SQLite 数据库（17 张表）+ 异步任务队列 (D1-as-Queue) |
| Cloudflare R2          | 对象存储（封面、用户文件）                  |

## 六、新增数据库表设计

### 6.1 积分相关表

```sql
-- 积分余额表
CREATE TABLE credit_balances (
  user_id TEXT PRIMARY KEY,
  balance INTEGER NOT NULL DEFAULT 0,     -- 当前余额
  total_earned INTEGER NOT NULL DEFAULT 0, -- 累计获得
  total_spent INTEGER NOT NULL DEFAULT 0,  -- 累计消耗
  updated_at INTEGER NOT NULL
);

-- 积分交易记录
CREATE TABLE credit_transactions (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  type TEXT NOT NULL,            -- 'earn' | 'spend' | 'refund'
  amount INTEGER NOT NULL,       -- 正数=收入, 负数=支出
  balance_after INTEGER NOT NULL, -- 交易后余额
  source TEXT NOT NULL,          -- 'subscription' | 'topup' | 'bonus' | 'ai_call' | 'refund'
  reference_id TEXT,             -- 关联的订阅/任务/充值 ID
  description TEXT,
  created_at INTEGER NOT NULL
);

-- 订阅表
CREATE TABLE subscriptions (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL UNIQUE,
  stripe_subscription_id TEXT,
  stripe_customer_id TEXT,
  plan TEXT NOT NULL DEFAULT 'free', -- 'free' | 'pro'
  status TEXT NOT NULL DEFAULT 'active', -- 'active' | 'canceled' | 'past_due' | 'trialing'
  current_period_start INTEGER,
  current_period_end INTEGER,
  monthly_credits INTEGER NOT NULL DEFAULT 200,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);

-- AI 模型积分消耗配置
CREATE TABLE model_pricing (
  id TEXT PRIMARY KEY,
  provider TEXT NOT NULL,        -- 'openrouter' | 'deepseek' | 'gemini' | 'kling'
  model_id TEXT NOT NULL,        -- 模型标识
  model_name TEXT NOT NULL,      -- 显示名称
  credits_per_call INTEGER NOT NULL, -- 每次调用消耗积分
  category TEXT NOT NULL,        -- 'text' | 'image' | 'video' | 'audio'
  is_active INTEGER DEFAULT 1,
  UNIQUE(provider, model_id)
);
```

### 6.2 异步任务表

```sql
-- 异步任务表
CREATE TABLE async_tasks (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  workflow_id TEXT,
  node_id TEXT,
  type TEXT NOT NULL,            -- 'video_gen' | 'image_gen' | 'audio_gen'
  status TEXT NOT NULL DEFAULT 'pending', -- 'pending' | 'running' | 'completed' | 'failed'
  provider TEXT NOT NULL,
  model_id TEXT NOT NULL,
  input_data TEXT NOT NULL,      -- JSON: 输入参数
  output_data TEXT,              -- JSON: 输出结果 (URL 等)
  error_message TEXT,
  credits_charged INTEGER DEFAULT 0,
  progress INTEGER DEFAULT 0,    -- 0-100
  external_task_id TEXT,         -- 第三方 API 的任务 ID
  created_at INTEGER NOT NULL,
  started_at INTEGER,
  completed_at INTEGER
);
```

## 七、新增 API 路由

### 7.1 积分与支付

```
/api/credits
├── GET    /balance           # 获取积分余额
├── GET    /transactions      # 积分交易历史（分页）
└── GET    /usage             # 使用统计（图表数据）

/api/subscriptions
├── GET    /current           # 当前订阅信息
├── POST   /create            # 创建订阅（Stripe Checkout）
├── POST   /cancel            # 取消订阅
└── POST   /change-plan       # 变更套餐

/api/topup
└── POST   /create            # 创建积分充值（Stripe Checkout）

/api/webhooks
├── POST   /stripe            # Stripe Webhook（订阅/支付事件）
└── POST   /clerk             # Clerk Webhook（用户事件）
```

### 7.2 AI 执行

```
/api/ai
├── POST   /execute           # 同步 AI 执行（文本生成等）
├── POST   /execute/stream    # 流式 AI 执行（LLM 对话）
├── POST   /task              # 提交异步任务（视频/图片生成）
├── GET    /task/:id          # 查询任务状态
└── GET    /models            # 获取可用模型列表（含积分价格）
```

### 7.3 定价

```
/api/pricing
└── GET    /plans             # 获取套餐列表（含功能矩阵）
```
