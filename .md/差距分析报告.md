## 一、架构决策记录

| 编号    | 决策项      | 选择                                  | 理由                                                              |
| ------- | ----------- | ------------------------------------- | ----------------------------------------------------------------- |
| ADR-001 | AI 执行模式 | 双模式并存（积分制 + 用户自带 Key）   | 积分制对标 martini.art 商业模式，自带 Key 降低用户门槛            |
| ADR-002 | 前端框架    | Next.js (App Router)                  | SSG/SSR/CSR 混合渲染，SEO 友好，Clerk 官方 SDK，部署到 Cloudflare |
| ADR-003 | 国际化      | P1 阶段中英双语 (next-intl)           | 从架构层面内建 i18n，避免后期大规模重构                           |
| ADR-004 | 支付系统    | Stripe                                | 国际主流，支持订阅 + 一次性支付，martini.art 同款                 |
| ADR-005 | 后端        | Cloudflare Workers + D1 + R2 + Queues | Serverless 全家桶，成本可控                                       |
| ADR-006 | 认证        | Clerk (Next.js SDK)                   | 开箱即用，支持 Webhook 同步                                       |

## 三、致命缺失详细分析

### 3.1 积分计费系统

**现状**：文档 P3 模糊提到"VIP 系统"，无任何架构设计。

**需要补充**：

- 积分数据模型（用户余额、消耗记录、充值记录）
- 套餐设计（Free / Standard / Pro / Ultimate）
- 不同 AI 模型的积分消耗表
- 积分扣费的事务安全（先扣后调、失败回滚）
- Stripe 订阅集成（月付/年付）
- 积分包一次性购买
- 用量统计与报表

**影响**：无计费系统 = 无法变现 = 项目不可持续

### 3.2 服务端 AI 执行架构

**现状**：设计为"用户自带 API Key，前端直调 AI API"。

**需要补充**（双模式并存）：

- 模式一：平台积分 → 后端 Worker 代理调用 → 平台 API Key → 扣积分
- 模式二：用户自带 Key → 后端 Worker 代理调用 → 用户 API Key → 不扣积分
- AI Provider 抽象层（统一 OpenRouter / Gemini / DeepSeek / Kling 等）
- 请求路由策略（根据执行模式选择 API Key）
- 流式响应代理（SSE 透传）
- 超时和重试策略

**影响**：商业模式的技术基座

### 3.3 异步任务队列

**现状**：只提了一句"异步任务轮询"。

**需要补充**：

- Cloudflare Queues 作为任务队列
- 任务生命周期：pending → running → completed / failed
- 任务状态数据模型
- 长时间任务（视频生成）的进度追踪
- 客户端轮询 / SSE 推送策略
- 失败重试与死信队列
- 并发限制（按用户套餐）

**影响**：视频/图片等非实时 AI 任务的必要基础设施

### 3.4 定价页面与套餐设计

**现状**：只有一个 `/pricing` 路由占位。

**需要补充**：

- 套餐层级设计（参考 martini.art 四档）
- 功能权限矩阵（模型访问、并发数、存储空间等）
- 年付折扣策略
- 积分包追加购买
- 免费试用策略（首次注册赠送积分）

## 四、重要缺失详细分析

### 4.1 速率限制与防滥用

- API 全局速率限制（基于 Cloudflare Rate Limiting）
- 按用户套餐的差异化限速
- AI API 调用并发限制
- 防刷策略（短时间大量注册、薅免费积分）

### 4.2 生成内容存储策略

- R2 存储结构设计（按用户/日期/类型分桶）
- CDN 分发策略（Cloudflare CDN 自动加速）
- 过期清理策略（免费用户生成内容 30 天过期？）
- 存储配额（按套餐限制）
- 缩略图自动生成（用于工作流卡片预览）

### 4.3 错误监控与日志

- Sentry 集成（前端 + Worker）
- 结构化日志（Cloudflare Logpush）
- 关键指标看板（API 响应时间、错误率、AI 调用成功率）

### 4.4 响应式设计策略

- 移动端适配（Landing Page 必须响应式）
- 画布编辑器：桌面端专属，移动端提示"请使用电脑访问"
- 作品广场 / 个人页面：响应式布局

### 4.5 法律页面

- `/terms` 服务条款
- `/privacy` 隐私政策
- `/refund` 退款政策（积分制退款规则）

## 五、框架迁移影响（Vite → Next.js）

### 5.1 项目结构重构

| 变化     | 旧 (Vite)                 | 新 (Next.js)                              |
| -------- | ------------------------- | ----------------------------------------- |
| 路由     | React Router (手动配置)   | App Router (文件系统路由)                 |
| 页面位置 | `src/pages/` 组件         | `app/` 目录                               |
| API 路由 | 独立 Worker               | Next.js Route Handlers + Worker (AI代理)  |
| 渲染     | 纯 CSR                    | SSG (Landing) + SSR (广场) + CSR (画布)   |
| 入口     | `main.tsx` + `index.html` | `app/layout.tsx`                          |
| 部署     | Cloudflare Pages (静态)   | Cloudflare Pages (@opennextjs/cloudflare) |
| Clerk    | `@clerk/clerk-react`      | `@clerk/nextjs` (功能更全)                |
| 中间件   | Worker 内 auth.ts         | `middleware.ts` (边缘运行)                |
| i18n     | 无                        | `next-intl`                               |

### 5.2 保留不变的部分

- ReactFlow 画布引擎
- Zustand 状态管理
- Tailwind CSS
- shadcn/ui 组件
- D1 / R2 / Cloudflare 基础设施
- 多模态类型系统

### 5.3 Worker 职责变化

**旧设计**：Worker = 全部后端 API
**新设计**：Worker 仅负责以下职责（Next.js Route Handlers 处理其余）

| 组件                   | 职责                                        |
| ---------------------- | ------------------------------------------- |
| Next.js Route Handlers | CRUD API、用户管理、积分管理、Webhook       |
| Cloudflare Worker      | AI API 代理（长时间流式请求）、异步任务消费 |
| Cloudflare Queues      | 异步任务队列（视频/图片生成）               |

## 六、新增数据库表设计

### 6.1 积分相关表

```sql
-- 积分余额表
CREATE TABLE credit_balances (
  user_id TEXT PRIMARY KEY,
  balance INTEGER NOT NULL DEFAULT 0,     -- 当前余额
  total_earned INTEGER NOT NULL DEFAULT 0, -- 累计获得
  total_spent INTEGER NOT NULL DEFAULT 0,  -- 累计消耗
  updated_at INTEGER NOT NULL
);

-- 积分交易记录
CREATE TABLE credit_transactions (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  type TEXT NOT NULL,            -- 'earn' | 'spend' | 'refund'
  amount INTEGER NOT NULL,       -- 正数=收入, 负数=支出
  balance_after INTEGER NOT NULL, -- 交易后余额
  source TEXT NOT NULL,          -- 'subscription' | 'topup' | 'bonus' | 'ai_call' | 'refund'
  reference_id TEXT,             -- 关联的订阅/任务/充值 ID
  description TEXT,
  created_at INTEGER NOT NULL
);

-- 订阅表
CREATE TABLE subscriptions (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL UNIQUE,
  stripe_subscription_id TEXT,
  stripe_customer_id TEXT,
  plan TEXT NOT NULL DEFAULT 'free', -- 'free' | 'standard' | 'pro' | 'ultimate'
  status TEXT NOT NULL DEFAULT 'active', -- 'active' | 'canceled' | 'past_due'
  current_period_start INTEGER,
  current_period_end INTEGER,
  monthly_credits INTEGER NOT NULL DEFAULT 200,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);

-- AI 模型积分消耗配置
CREATE TABLE model_pricing (
  id TEXT PRIMARY KEY,
  provider TEXT NOT NULL,        -- 'openrouter' | 'deepseek' | 'gemini' | 'kling'
  model_id TEXT NOT NULL,        -- 模型标识
  model_name TEXT NOT NULL,      -- 显示名称
  credits_per_call INTEGER NOT NULL, -- 每次调用消耗积分
  category TEXT NOT NULL,        -- 'text' | 'image' | 'video' | 'audio'
  is_active INTEGER DEFAULT 1,
  UNIQUE(provider, model_id)
);
```

### 6.2 异步任务表

```sql
-- 异步任务表
```

## 七、新增 API 路由

### 7.1 积分与支付

```
/api/credits
├── GET    /balance           # 获取积分余额
├── GET    /transactions      # 积分交易历史（分页）
└── GET    /usage             # 使用统计（图表数据）

/api/subscriptions
├── GET    /current           # 当前订阅信息
├── POST   /create            # 创建订阅（Stripe Checkout）
├── POST   /cancel            # 取消订阅
└── POST   /change-plan       # 变更套餐

/api/topup
└── POST   /create            # 创建积分充值（Stripe Checkout）

/api/webhooks
├── POST   /stripe            # Stripe Webhook（订阅/支付事件）
└── POST   /clerk             # Clerk Webhook（用户事件）
```

### 7.2 AI 执行

```
/api/ai
├── POST   /execute           # 同步 AI 执行（文本生成等）
├── POST   /execute/stream    # 流式 AI 执行（LLM 对话）
├── POST   /task              # 提交异步任务（视频/图片生成）
├── GET    /task/:id          # 查询任务状态
└── GET    /models            # 获取可用模型列表（含积分价格）
```

```
/api/pricing
└── GET    /plans             # 获取套餐列表（含功能矩阵）
```
