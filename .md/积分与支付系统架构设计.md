## 一、概述

### 1.1 设计目标

构建完整的积分计费与 Stripe 支付系统，实现平台商业化闭环。系统支持**双模式 AI 执行**（平台积分 + 用户自带 API Key），对标 martini.art 的四档订阅定价模型。

### 1.2 核心价值

1. **商业可持续**：积分制变现是平台存活的血液，没有计费系统的 AI 平台等于烧钱慈善
2. **用户友好**：双模式并存，低门槛（自带 Key 免费用）+ 高便利（积分一键调用）
3. **事务安全**：先冻结后扣费，失败自动退还，积分不会凭空消失也不会超额消耗
4. **架构简洁**：Stripe 处理支付复杂度，D1 处理积分账本，Worker 处理 AI 代理，各司其职

## 三、积分系统架构

### 3.1 积分余额管理

每个用户维护三个核心指标：

| 字段 | 含义 | 用途 |
|------|------|------|
| `balance` | 当前可用余额 | 判断是否可发起 AI 调用 |
| `total_earned` | 累计获得积分 | 用户统计展示 |
| `total_spent` | 累计消耗积分 | 用户统计展示 |

```typescript
interface CreditBalance {
  userId: string
  balance: number       // 当前可用 (含冻结部分已扣除)
  frozen: number        // 冻结中 (正在执行的 AI 调用)
  totalEarned: number   // 累计获得
  totalSpent: number    // 累计消耗
  updatedAt: number     // 最后更新时间戳
}
```

### 3.2 积分获取方式

| 来源 | 触发条件 | 积分数 | 有效期 |
|------|---------|--------|--------|
| 订阅月充 | 订阅续费成功 (invoice.paid) | 按套餐等级 | 当月有效，月底清零 |
| 积分包购买 | 一次性支付成功 | 按购买数量 | 永久有效 |
| 新注册赠送 | 首次创建账户 | 200 | 永久有效 |
| 邀请奖励 | 被邀请用户完成注册 | 100 | 永久有效 |

### 3.3 积分有效期策略

采用**双池模型**：

```
┌─────────────────────────────────────┐
│          用户积分余额                │
│                                     │
│  ┌──────────────┐ ┌──────────────┐  │
│  │  订阅积分池   │ │  永久积分池   │  │
│  │  (monthly)   │ │  (permanent) │  │
│  │              │ │              │  │
│  │ 每月重置为   │ │ 购买/赠送/   │  │
│  │ 套餐额度     │ │ 邀请获得     │  │
│  │ 不累积       │ │ 永不过期     │  │
│  └──────────────┘ └──────────────┘  │
│                                     │
│  消耗优先级: 订阅积分 → 永久积分     │
└─────────────────────────────────────┘
```

**关键规则**：
- 订阅积分每月重置（不累积），未用完的作废
- 永久积分（购买/赠送/邀请）永不过期
- 消耗时优先使用订阅积分，再使用永久积分
- 余额 = `monthly_balance` + `permanent_balance`

### 3.5 事务安全：冻结-扣费-退还

这是积分系统的核心设计，必须保证**不丢积分、不超扣积分**。

```
用户发起 AI 调用
       │
       ▼
┌─────────────────┐
│ 1. 检查余额     │ ← balance >= required_credits ?
│    余额不足？    │    否 → 返回 402 余额不足
└───────┬─────────┘
        │ 是
        ▼
┌─────────────────┐
│ 2. 冻结积分     │ ← balance -= credits, frozen += credits
│    记录冻结事务  │    transaction.type = 'freeze'
└───────┬─────────┘
        │
        ▼
┌─────────────────┐
│ 3. 执行 AI 调用 │ ← 调用 AI Provider API
└───────┬─────────┘
        │
   ┌────┴────┐
   │         │
  成功      失败
   │         │
   ▼         ▼
┌──────┐  ┌──────────┐
│ 4a.  │  │ 4b.      │
│ 确认 │  │ 退还     │
│ 扣费 │  │ 积分     │
│      │  │          │
│frozen│  │frozen    │
│-=    │  │-= credits│
│spent │  │balance   │
│+=    │  │+= credits│
└──────┘  └──────────┘
```

```typescript
// ================================================================
// 积分事务核心实现
// 使用 D1 事务保证原子性
// ================================================================

type TransactionType = 'earn' | 'spend' | 'freeze' | 'unfreeze' | 'refund'

interface CreditTransaction {
  id: string
  userId: string
  type: TransactionType
  amount: number          // 正数 = 收入/退还, 负数 = 消耗/冻结
  balanceAfter: number    // 事务后可用余额
  source: string          // 'subscription' | 'topup' | 'bonus' | 'invite' | 'ai_call' | 'refund'
  referenceId?: string    // 关联的任务/订单 ID
  description?: string
  createdAt: number
}

async function freezeCredits(
  db: D1Database,
  userId: string,
  amount: number,
  referenceId: string
): Promise<{ success: boolean; error?: string }> {
  // D1 不支持真正的事务, 使用 batch 保证原子性
  const balance = await db
    .prepare('SELECT balance, frozen FROM credit_balances WHERE user_id = ?')
    .bind(userId)
    .first<{ balance: number; frozen: number }>()

  if (!balance || balance.balance < amount) {
    return { success: false, error: 'INSUFFICIENT_CREDITS' }
  }

  const newBalance = balance.balance - amount
  const newFrozen = balance.frozen + amount
  const txId = crypto.randomUUID()

  await db.batch([
    db.prepare(
      'UPDATE credit_balances SET balance = ?, frozen = ?, updated_at = ? WHERE user_id = ? AND balance >= ?'
    ).bind(newBalance, newFrozen, Date.now(), userId, amount),

    db.prepare(
      `INSERT INTO credit_transactions (id, user_id, type, amount, balance_after, source, reference_id, created_at)
       VALUES (?, ?, 'freeze', ?, ?, 'ai_call', ?, ?)`
    ).bind(txId, userId, -amount, newBalance, referenceId, Date.now()),
  ])

  return { success: true }
}

async function confirmSpend(
  db: D1Database,
  userId: string,
  amount: number,
  referenceId: string
): Promise<void> {
  const txId = crypto.randomUUID()

  await db.batch([
    db.prepare(
      'UPDATE credit_balances SET frozen = frozen - ?, total_spent = total_spent + ?, updated_at = ? WHERE user_id = ?'
    ).bind(amount, amount, Date.now(), userId),

    db.prepare(
      `INSERT INTO credit_transactions (id, user_id, type, amount, balance_after, source, reference_id, created_at)
       VALUES (?, ?, 'spend', ?, (SELECT balance FROM credit_balances WHERE user_id = ?), 'ai_call', ?, ?)`
    ).bind(txId, userId, -amount, userId, referenceId, Date.now()),
  ])
}

async function refundCredits(
  db: D1Database,
  userId: string,
  amount: number,
  referenceId: string
): Promise<void> {
  const txId = crypto.randomUUID()

  await db.batch([
    db.prepare(
      'UPDATE credit_balances SET balance = balance + ?, frozen = frozen - ?, updated_at = ? WHERE user_id = ?'
    ).bind(amount, amount, Date.now(), userId),

    db.prepare(
      `INSERT INTO credit_transactions (id, user_id, type, amount, balance_after, source, reference_id, created_at)
       VALUES (?, ?, 'refund', ?, (SELECT balance FROM credit_balances WHERE user_id = ?), 'ai_call', ?, ?)`
    ).bind(txId, userId, amount, userId, referenceId, Date.now()),
  ])
}
```

### 4.1 架构总览

```
                    用户请求 (POST /api/ai/execute)
                              │
                              ▼
                    ┌──────────────────┐
                    │ 读取执行模式      │
                    │ (节点级 > 用户级) │
                    └────────┬─────────┘
                             │
                  ┌──────────┴──────────┐
                  │                     │
            模式一: 积分            模式二: 自带 Key
                  │                     │
                  ▼                     ▼
         ┌────────────────┐    ┌────────────────┐
         │ 验证余额        │    │ 解密用户 Key    │
         │ 冻结积分        │    │ (AES-256-GCM)  │
         └───────┬────────┘    └───────┬────────┘
                 │                     │
                 ▼                     ▼
         ┌────────────────────────────────────┐
         │     AI Provider 统一抽象层          │
         │  ┌──────┐ ┌──────┐ ┌──────┐       │
         │  │OpenR.│ │Gemini│ │Kling │ ...   │

                          │
                     ┌────┴────┐
                    成功      失败
                     │         │
                     ▼         ▼
              确认扣费     退还冻结积分
              (积分模式)   (积分模式)
```

### 4.2 模式一：积分模式

用户使用平台积分，后端使用平台 API Key 代理调用。

```typescript
// ================================================================
// 积分模式执行流程
// ================================================================

async function executeWithCredits(
  ctx: ExecutionContext,
  request: AIExecuteRequest
): Promise<AIExecuteResponse> {
  const { userId, provider, modelId, input } = request

  // --- Step 1: 查询模型定价 ---
  const pricing = await getModelPricing(ctx.db, provider, modelId)
  if (!pricing) throw new AppError('MODEL_NOT_FOUND', 404)

  // --- Step 2: 冻结积分 ---
  const freezeResult = await freezeCredits(ctx.db, userId, pricing.creditsPerCall, request.taskId)
  if (!freezeResult.success) {
    throw new AppError('INSUFFICIENT_CREDITS', 402)
  }

  try {
    // --- Step 3: 使用平台 Key 调用 AI ---
    const aiProvider = getAIProvider(provider)
    const result = await aiProvider.execute({
      modelId,
      input,
      apiKey: ctx.env[`${provider.toUpperCase()}_API_KEY`], // 平台 Key
    })

    // --- Step 4a: 成功 → 确认扣费 ---
    await confirmSpend(ctx.db, userId, pricing.creditsPerCall, request.taskId)

    return result
  } catch (error) {
    // --- Step 4b: 失败 → 退还积分 ---
    await refundCredits(ctx.db, userId, pricing.creditsPerCall, request.taskId)
    throw error
  }
}
```

### 4.3 模式二：自带 Key 模式

用户使用自己的 API Key，不消耗平台积分。后端仍然代理请求（不暴露前端直调）。

```typescript
// ================================================================
// 自带 Key 模式执行流程
// ================================================================

async function executeWithUserKey(
  ctx: ExecutionContext,
  request: AIExecuteRequest
): Promise<AIExecuteResponse> {
  const { userId, provider, modelId, input } = request

  // --- Step 1: 获取并解密用户 Key ---
  const encryptedKey = await getUserAPIKey(ctx.db, userId, provider)
  if (!encryptedKey) {
    throw new AppError('API_KEY_NOT_CONFIGURED', 400)
  }
  const apiKey = await decryptAPIKey(encryptedKey, ctx.env.ENCRYPTION_KEY)

  // --- Step 2: 使用用户 Key 调用 AI (不扣积分) ---
  const aiProvider = getAIProvider(provider)
  const result = await aiProvider.execute({
    modelId,
    input,
    apiKey,
  })

  // --- Step 3: 记录使用日志 (不扣积分, 仅统计) ---
  await logUsage(ctx.db, userId, provider, modelId, 'user_key')

  return result
}
```

### 4.4 模式选择优先级

```
节点级配置 (node.data.executionMode)
       │
```

### 4.5 API Key 加密存储方案

使用 AES-256-GCM 对称加密，密钥存储在 Worker 环境变量中。

```typescript
// ================================================================
// API Key 加密/解密
// 使用 Web Crypto API (Cloudflare Workers 原生支持)
// ================================================================

async function encryptAPIKey(
  plainKey: string,
  encryptionKey: string
): Promise<string> {
  const encoder = new TextEncoder()
  const iv = crypto.getRandomValues(new Uint8Array(12))

  const key = await crypto.subtle.importKey(
    'raw',
    hexToBuffer(encryptionKey),
    { name: 'AES-GCM' },
    false,
    ['encrypt']
  )

  const encrypted = await crypto.subtle.encrypt(
    { name: 'AES-GCM', iv },
    key,
    encoder.encode(plainKey)
  )

  // 格式: iv(hex):ciphertext(hex)
  return bufferToHex(iv) + ':' + bufferToHex(new Uint8Array(encrypted))
}

async function decryptAPIKey(
  encryptedKey: string,
  encryptionKey: string
): Promise<string> {
  const [ivHex, ciphertextHex] = encryptedKey.split(':')
  const iv = hexToBuffer(ivHex)
  const ciphertext = hexToBuffer(ciphertextHex)

  const key = await crypto.subtle.importKey(
    'raw',
    hexToBuffer(encryptionKey),
    { name: 'AES-GCM' },
    false,
    ['decrypt']
  )

  const decrypted = await crypto.subtle.decrypt(
    { name: 'AES-GCM', iv },
    key,
    ciphertext
  )

  return new TextDecoder().decode(decrypted)
}
```

## 五、Stripe 集成

### 5.1 订阅创建流程 (Stripe Checkout)

```
用户点击 "升级到 Pro"
       │
       ▼
前端: POST /api/billing/checkout
       │
       ▼
后端:
  1. 获取或创建 Stripe Customer (关联 Clerk userId)
  2. 创建 Stripe Checkout Session
     - mode: 'subscription'
     - line_items: [{ price: 'price_pro_monthly', quantity: 1 }]
     - success_url: '/billing?success=true'
     - cancel_url: '/pricing'
     - metadata: { userId, plan: 'pro' }
  3. 返回 session.url
       │
       ▼
前端: window.location.href = session.url
       │
       ▼
Stripe Checkout 页面 (Stripe 托管)
       │
       ▼
支付成功 → 重定向到 /billing?success=true
       │
       ▼ (异步)
Stripe Webhook → checkout.session.completed
       │
       ▼
后端:
  1. 获取 subscription 信息
  2. 写入 subscriptions 表
  3. 充值对应积分到 credit_balances
```

```typescript
// ================================================================
// 创建 Stripe Checkout Session
// ================================================================

import Stripe from 'stripe'

async function createCheckoutSession(
  stripe: Stripe,
  db: D1Database,
  userId: string,
  plan: 'standard' | 'pro' | 'ultimate',
  billingPeriod: 'monthly' | 'yearly'
): Promise<string> {
  // --- 获取或创建 Stripe Customer ---
  let customer = await getStripeCustomer(db, userId)
  if (!customer) {
    const stripeCustomer = await stripe.customers.create({
      metadata: { userId },
    })
    await saveStripeCustomer(db, userId, stripeCustomer.id)
    customer = stripeCustomer.id
  }

  // --- 创建 Checkout Session ---
  const priceId = STRIPE_PRICES[plan][billingPeriod]
  const session = await stripe.checkout.sessions.create({
    customer,
    mode: 'subscription',
    line_items: [{ price: priceId, quantity: 1 }],
    success_url: `${process.env.NEXT_PUBLIC_URL}/billing?success=true`,
    cancel_url: `${process.env.NEXT_PUBLIC_URL}/pricing`,
    metadata: { userId, plan, billingPeriod },
    subscription_data: {
      metadata: { userId, plan },
    },
  })

  return session.url!
}
```

### 5.2 Stripe Webhook 事件处理

```typescript
// ================================================================
// Stripe Webhook 处理器
// 所有积分充值、订阅状态变更都通过 Webhook 驱动
// ================================================================

type StripeWebhookHandler = (event: Stripe.Event, db: D1Database) => Promise<void>

const webhookHandlers: Record<string, StripeWebhookHandler> = {
  // --- 订阅首次创建成功 ---
  'checkout.session.completed': async (event, db) => {
    const session = event.data.object as Stripe.Checkout.Session
    const { userId, plan } = session.metadata!

    if (session.mode === 'subscription') {
      // 订阅类型: 创建订阅记录 + 充值积分
      const subscription = await stripe.subscriptions.retrieve(
        session.subscription as string
      )
      await createSubscription(db, userId, plan, subscription)
      await addCredits(db, userId, PLAN_CREDITS[plan], 'subscription', subscription.id)
    } else if (session.mode === 'payment') {
      // 一次性支付: 积分包购买
      const { creditAmount } = session.metadata!
      await addCredits(db, userId, parseInt(creditAmount), 'topup', session.id)
    }
  },

  // --- 订阅续费成功 (每月/每年自动扣款) ---
  'invoice.paid': async (event, db) => {
    const invoice = event.data.object as Stripe.Invoice
    if (invoice.billing_reason === 'subscription_cycle') {
      const subscription = await stripe.subscriptions.retrieve(
        invoice.subscription as string
      )
      const { userId, plan } = subscription.metadata

      // 重置月度订阅积分 (不是累加, 是重置)
      await resetMonthlyCredits(db, userId, PLAN_CREDITS[plan])
    }
  },

  // --- 订阅取消 ---
  'customer.subscription.deleted': async (event, db) => {
    const subscription = event.data.object as Stripe.Subscription
    const { userId } = subscription.metadata

    await updateSubscription(db, userId, {
      status: 'canceled',
      plan: 'free',
      monthlyCredits: PLAN_CREDITS.free,
    })

    // 重置为免费套餐积分
    await resetMonthlyCredits(db, userId, PLAN_CREDITS.free)
  },

  // --- 订阅变更 (升级/降级) ---
  'customer.subscription.updated': async (event, db) => {
    const subscription = event.data.object as Stripe.Subscription
    const previous = event.data.previous_attributes as Partial<Stripe.Subscription>
    const { userId, plan } = subscription.metadata

    // 只在套餐实际变更时处理
    if (previous.items) {
      await updateSubscription(db, userId, {
        plan,
        monthlyCredits: PLAN_CREDITS[plan],
        status: subscription.status === 'active' ? 'active' : 'past_due',
      })
    }
  },

  // --- 支付失败 ---
  'invoice.payment_failed': async (event, db) => {
    const invoice = event.data.object as Stripe.Invoice
    const subscription = await stripe.subscriptions.retrieve(
      invoice.subscription as string
    )
    const { userId } = subscription.metadata

    await updateSubscription(db, userId, { status: 'past_due' })
    // 可选: 发送通知邮件
  },
}

// --- Webhook 入口 ---
async function handleStripeWebhook(
  request: Request,
  env: Env,
  db: D1Database
): Promise<Response> {
  const signature = request.headers.get('stripe-signature')!
  const body = await request.text()

  let event: Stripe.Event
  try {
    event = stripe.webhooks.constructEvent(body, signature, env.STRIPE_WEBHOOK_SECRET)
  } catch {
    return new Response('Invalid signature', { status: 400 })
  }

  const handler = webhookHandlers[event.type]
  if (handler) {
    await handler(event, db)
  }

  return new Response('OK', { status: 200 })
}
```

```typescript
async function createTopupCheckout(
  stripe: Stripe,
  db: D1Database,
  userId: string,
  packageId: string
): Promise<string> {
  const pack = CREDIT_PACKAGES.find(p => p.id === packageId)
  if (!pack) throw new AppError('INVALID_PACKAGE', 400)

  const customer = await getOrCreateStripeCustomer(stripe, db, userId)

  const session = await stripe.checkout.sessions.create({
    customer,
    mode: 'payment',
    line_items: [{
      price_data: {
        currency: 'usd',
        product_data: { name: pack.label },
        unit_amount: pack.price,
      },
      quantity: 1,
    }],
    success_url: `${process.env.NEXT_PUBLIC_URL}/billing?topup=success`,
    cancel_url: `${process.env.NEXT_PUBLIC_URL}/billing`,
    metadata: { userId, packageId, creditAmount: String(pack.credits) },
  })

  return session.url!
}
```

```typescript
async function handleRefund(

  userId: string,
  paymentIntentId: string,

): Promise<void> {

  }
```

### 8.3 CreditBalance - 积分余额组件

嵌入在顶部栏或侧边栏的轻量组件，实时显示积分余额。

```typescript
// ================================================================
// 积分余额组件 (侧边栏/顶部栏内嵌)
// ================================================================

function CreditBalance() {
  const { data: balance } = useQuery({
    queryKey: ['credits', 'balance'],
    queryFn: () => api.get('/api/credits/balance'),
    refetchInterval: 30_000, // 30秒自动刷新
  })

  if (!balance) return <Skeleton className="h-8 w-24" />

  const total = balance.monthlyBalance + balance.permanentBalance
  const percentage = balance.plan !== 'free'
    ? (balance.monthlyBalance / balance.monthlyCredits) * 100
    : (total / 200) * 100

  return (
    <div className="flex items-center gap-2 px-3 py-1.5 rounded-lg bg-gray-50">
      <Zap className="w-4 h-4 text-yellow-500" />
      <span className="text-sm font-medium">{total.toLocaleString()}</span>
      <div className="w-16 h-1.5 bg-gray-200 rounded-full overflow-hidden">
        <div
          className="h-full bg-yellow-500 rounded-full transition-all"
          style={{ width: `${Math.min(percentage, 100)}%` }}
        />
      </div>
    </div>
  )
}
```

### 9.2 并发扣费的原子性

```typescript
// ================================================================
// D1 batch 保证积分操作的原子性
// 所有读-判断-写操作通过 batch 执行
// ================================================================

// 关键: 使用 WHERE balance >= amount 的条件更新
// 如果余额不足, UPDATE 影响行数为 0, 整个 batch 无效果

async function atomicFreezeCredits(
  db: D1Database,
  userId: string,
  amount: number
): Promise<boolean> {
  const results = await db.batch([
    // 条件更新: 只有余额充足时才会生效
    db.prepare(
      `UPDATE credit_balances
       SET monthly_balance = CASE
         WHEN monthly_balance >= ? THEN monthly_balance - ?
         ELSE monthly_balance
       END,
       permanent_balance = CASE
         WHEN monthly_balance < ? THEN permanent_balance - (? - monthly_balance)
         ELSE permanent_balance
       END,
       frozen = frozen + ?,
       updated_at = ?
       WHERE user_id = ?
       AND (monthly_balance + permanent_balance) >= ?`
    ).bind(amount, amount, amount, amount, amount, Date.now(), userId, amount),
  ])

  // 检查是否真正更新了行
  return (results[0] as D1Result).meta.changes > 0
}
```

### 9.3 Stripe Webhook 签名验证

```typescript
// ================================================================
// Stripe Webhook 安全验证
// 必须验证签名, 防止伪造 Webhook
// ================================================================

async function verifyStripeSignature(
  body: string,
  signature: string,
  secret: string
): Promise<Stripe.Event> {
  try {
    return stripe.webhooks.constructEvent(body, signature, secret)
  } catch (err) {
    throw new AppError('INVALID_WEBHOOK_SIGNATURE', 400)
  }
}

// 幂等性保证: 通过 event.id 去重
// Stripe 可能重复发送同一事件
async function isEventProcessed(db: D1Database, eventId: string): Promise<boolean> {
  const row = await db
    .prepare('SELECT id FROM processed_stripe_events WHERE event_id = ?')
    .bind(eventId)
    .first()
  return !!row
}

async function markEventProcessed(db: D1Database, eventId: string): Promise<void> {
  await db
    .prepare('INSERT OR IGNORE INTO processed_stripe_events (event_id, processed_at) VALUES (?, ?)')
    .bind(eventId, Date.now())
    .run()
}
```

### 9.4 API Key 加密存储安全

```
安全层级:
1. 传输层: HTTPS 加密 (Cloudflare 自动)
2. 存储层: AES-256-GCM 加密 (server-side)
3. 密钥管理: 加密密钥存储在 Worker Secrets (wrangler secret put ENCRYPTION_KEY)
4. 访问控制: 只有用户本人可以管理自己的 Key
5. 展示层: API Key 永远不返回明文, 仅返回掩码 (sk-****abcd)
6. 日志脱敏: 任何日志中不得出现 API Key 明文
```
