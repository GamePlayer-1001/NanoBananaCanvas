# 商业化移除清单

> 目标：将当前项目从“具备订阅/积分/Stripe 依赖的商业化架构”收敛为“无付费、无套餐门槛、画板优先”的可持续开发状态。

## 一、拆除目标

1. 移除所有面向用户的付费入口。
2. 移除所有 Stripe 运行时依赖与 webhook 入口。
3. 移除套餐、积分、订阅对编辑器与任务执行的控制。
4. 保留未来重建商业化所需的最小知识文档，不保留脏耦合实现。

## 二、拆除范围

### 2.1 前端 UI

- `apps/web/app/[locale]/(landing)/pricing/page.tsx`
- `apps/web/components/pricing/*`
- `apps/web/components/profile/billing-tab.tsx`
- `apps/web/components/profile/subscription-tab.tsx`
- `apps/web/components/profile/payment-history.tsx`
- `apps/web/components/layout/app-sidebar.tsx` 中的升级入口、价格弹窗入口、积分展示
- `apps/web/messages/en.json` 与 `apps/web/messages/zh.json` 中的 pricing/billing/subscription/credits 文案

### 2.2 Hook 与 Query 层

- `apps/web/hooks/use-billing.ts`
- `apps/web/hooks/use-user.ts` 中仅服务商业化的余额/订阅查询
- `apps/web/lib/query/keys.ts` 中 `billing`、`credits` 相关 key

### 2.3 API 路由

- `apps/web/app/api/billing/*`
- `apps/web/app/api/credits/*`
- `apps/web/app/api/webhooks/stripe/route.ts`

### 2.4 服务与领域逻辑

- `apps/web/lib/stripe.ts`
- `apps/web/lib/credits/*`
- `apps/web/lib/validations/billing.ts`
- 任务与 AI 执行中所有套餐/积分分支

### 2.5 数据与配置

- `apps/web/db/schema.sql` 中 `credit_balances`、`credit_transactions`、`subscriptions`、`model_pricing`、`processed_stripe_events`
- `apps/web/db/seed-pricing.sql`
- 与 Stripe / pricing 相关 migration
- `packages/shared/src/constants/index.ts` 中 `PLANS`、`PRO_PRICING` 若仍只服务商业化，需要重构或移除

## 三、建议保留项

1. 保留异步任务系统本身，但去掉 `execution_mode = credits` 的收费语义。
2. 保留模型目录能力，但移除“套餐决定模型访问范围”的限制。
3. 保留相关历史设计文档，作为未来重建时的参考输入。

## 四、执行顺序

1. 先移除前端可见入口，避免用户再进入坏路径。
2. 再切掉 API 与 hook，防止前端残余请求打到废接口。
3. 再改执行链，去掉积分冻结、套餐判断、支付回调。
4. 最后清数据库 schema、种子与环境变量引用。

## 五、关键风险

1. `app-sidebar.tsx`、`profile-modal.tsx` 可能因移除 tab 后出现 UI 索引错位。
2. `ai/execute`、`ai/stream`、`tasks/service` 依赖 `credits` 模块，不能只删 API，不改执行链。
3. `schema.sql` 与现网 D1 不会自动同步，代码删完后仍需规划本地/远端迁移策略。
4. `messages/*.json` 体量大，容易留下无消费文案与丢 key 的双向污染。

## 六、验收清单

- [ ] 项目中不再存在 `/pricing` 页面入口
- [ ] 侧边栏、ProfileModal 中不再出现升级/账单/订阅/积分入口
- [ ] `pnpm lint` 不再引用已删除的 billing/credits/pricing 模块
- [ ] `pnpm test` 不再包含失效的商业化测试
- [ ] `rg "stripe|billing|credits|subscription"` 结果仅剩文档或明确保留的历史说明
- [ ] AI 执行与异步任务在无套餐、无积分前提下仍可运行

## 七、完成定义

完成不等于“页面删了”，而等于：

1. 用户看不到商业化 UI。
2. 运行时不再依赖 Stripe 与积分逻辑。
3. 数据模型不再以套餐/积分为单一真相源。
4. 后续重建商业化时，不需要先从废墟里辨认旧耦合。
