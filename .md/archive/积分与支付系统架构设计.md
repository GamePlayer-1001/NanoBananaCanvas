## 一、概述

### 1.1 设计目标

构建完整的积分计费与 Stripe 支付系统，实现平台商业化闭环。系统支持**双模式 AI 执行**（平台积分 + 用户自带 API Key），采用 **Free / Standard / Pro / Ultimate 四档套餐 + 积分包补充**的双链路商业化模型。

### 1.2 核心价值

1. **商业可持续**：积分制变现是平台存活的血液，没有计费系统的 AI 平台等于烧钱慈善
2. **用户友好**：双模式并存，低门槛（自带 Key 免费用）+ 高便利（平台积分一键调用）
3. **事务安全**：先冻结后扣费，失败自动退还，积分不会凭空消失也不会超额消耗
4. **架构简洁**：Stripe 处理支付复杂度与多币种展示，D1 处理积分账本，Worker 处理 AI 代理，各司其职

## 二、套餐设计

### 2.1 主方案选择

本次统一决策采用**四档套餐主方案**，理由如下：

1. 四档方案天然覆盖免费引流、标准付费、专业付费与高阶付费分层，后续扩展空间更稳定。
2. 积分包链路与四档权益更容易共存，避免二档方案中“所有付费用户都挤进同一档”的定价粗糙问题。
3. 二档方案中值得保留的部分，仅保留到实现层：Stripe Checkout/Portal 托管、Webhook 驱动、较少的运行时分支。

### 2.2 套餐链路与积分包链路

商业化拆成两条独立链路：

1. **套餐链路**
   Free / Standard / Pro / Ultimate 四档。
   支持 `auto_monthly` 自动月付订阅，以及 `one_time` 一次性套餐购买两种购买方式。
2. **积分包链路**
   面向需要临时补量的用户，单独购买永久积分，不改变当前套餐等级。

### 2.3 四档定价方案

| 套餐         | 自动月付 | 一次性套餐 | 基础积分 | 存储空间 |
| ------------ | -------- | ---------- | -------- | -------- |
| **Free**     | $0       | $0         | 0        | 1 GB     |
| **Standard** | $20      | $24        | 1,600    | 10 GB    |
| **Pro**      | $50      | $59        | 5,400    | 50 GB    |
| **Ultimate** | $150     | $179       | 17,000   | 200 GB   |

说明：

- `Free` 是默认进入态，不发基础积分；免费用户如需走平台 credits 模式，必须通过积分包获得余额。
- 一次性套餐本质是“按档位打包出售的非续费权益”，它与自动月付共享套餐等级，但不触发后续自动扣款。
- 当前套餐只影响两类权益：`积分额度` 与 `存储空间`。不再让套餐直接控制并发、模型访问层级或其他产品能力。

### 2.5 Stripe 多币种策略

支付网关仍使用 Stripe，但展示价格时增加区域价格选择层：

1. 服务端依据 `CF-IPCountry`、结账时的 IP 地域与已选择 locale 推断币种。
2. 每个套餐与积分包在 Stripe 中预建多币种 Price。
3. `/api/billing/checkout` 只接收业务语义参数：`plan`、`purchaseMode`、`currency`，实际映射到对应的 `price_id`。
4. 最终扣款金额以 Stripe Checkout 展示为准；本地只记录标准化金额与原始币种。

### 2.6 Free 默认进入态

`Free` 不是一个需要用户下单购买的真实收费 SKU，而是系统的默认进入态：

1. 新用户注册或匿名首次进入时，默认获得 `free` 计划身份。
2. `Free` 只提供基础存储额度，不发放平台积分。
3. 免费用户仍可通过 `user_key` 模式完整使用画板与 AI 节点，不被支付系统阻断。
4. 免费用户如要使用 `credits` 模式，必须先购买积分包，或升级为付费套餐。
5. 在 Stripe 侧不强制为 `Free` 创建独立 Checkout 流程，运行时只在本地 `subscriptions/users` 视图中表达其默认态。

## 三、积分系统架构

### 3.1 积分余额管理

每个用户维护三个核心指标：

| 字段           | 含义         | 用途                   |
| -------------- | ------------ | ---------------------- |
| `balance`      | 当前可用余额 | 判断是否可发起 AI 调用 |
| `total_earned` | 累计获得积分 | 用户统计展示           |
| `total_spent`  | 累计消耗积分 | 用户统计展示           |

```typescript
interface CreditBalance {
  userId: string
  balance: number // 当前可用 (含冻结部分已扣除)
  frozen: number // 冻结中 (正在执行的 AI 调用)
  totalEarned: number // 累计获得
  totalSpent: number // 累计消耗
  updatedAt: number // 最后更新时间戳
}
```

### 3.2 积分获取方式

| 来源       | 触发条件                    | 积分数     | 有效期             |
| ---------- | --------------------------- | ---------- | ------------------ |
| 订阅月充   | 自动月付续费成功 (invoice.paid) | 按套餐等级 | 当期有效，按周期重置 |
| 一次性套餐 | 一次性套餐支付成功              | 按套餐等级 | 当前套餐有效期内 |
| 积分包购买 | 一次性支付成功              | 按购买数量 | 永久有效           |
| 新注册赠送 | 首次创建账户                | 0          | -                  |
| 邀请奖励   | 被邀请用户完成注册          | 100        | 永久有效           |

### 3.3 积分有效期策略

采用**双池模型**：

```
┌─────────────────────────────────────┐
│          用户积分余额                │
│                                     │
│  ┌──────────────┐ ┌──────────────┐  │
│  │  订阅积分池   │ │  永久积分池   │  │
│  │  (monthly)   │ │  (permanent) │  │
│  │              │ │              │  │
│  │ 每月重置为   │ │ 购买/赠送/   │  │
│  │ 套餐额度     │ │ 邀请获得     │  │
│  │ 不累积       │ │ 永不过期     │  │
│  └──────────────┘ └──────────────┘  │
│                                     │
│  消耗优先级: 订阅积分 → 永久积分     │
└─────────────────────────────────────┘
```

**关键规则**：

- 订阅积分每月重置（不累积），未用完的作废
- 永久积分（购买/赠送/邀请）永不过期
- 消耗时优先使用订阅积分，再使用永久积分
- 余额 = `monthly_balance` + `permanent_balance`

### 3.4 Token 计费规则

历史按“每次调用固定积分”计费的方案过于粗糙，本次统一改为**按 token/生成量折算积分**：

1. 文本模型：按 `input_tokens + output_tokens` 折算。
2. 图片模型：按生成张数、分辨率与推理档位映射为等价 token 单位后折算。
3. 视频/音频模型：按时长、帧数或采样档位映射为等价 token 单位后折算。
4. 每个模型保留 `credits_per_1k_units` 或等价倍率，不再只存单次固定积分。

折算公式：

```typescript
credits = ceil((billableUnits / 1000) * creditsPer1kUnits)
```

### 3.5 事务安全：冻结-扣费-退还

这是积分系统的核心设计，必须保证**不丢积分、不超扣积分**。

```
用户发起 AI 调用
       │
       ▼
┌─────────────────┐
│ 1. 检查余额     │ ← balance >= required_credits ?
│    余额不足？    │    否 → 返回 402 余额不足
└───────┬─────────┘
        │ 是
        ▼
┌─────────────────┐
│ 2. 冻结积分     │ ← balance -= credits, frozen += credits
│    记录冻结事务  │    transaction.type = 'freeze'
└───────┬─────────┘
        │
        ▼
┌─────────────────┐
│ 3. 执行 AI 调用 │ ← 调用 AI Provider API
└───────┬─────────┘
        │
   ┌────┴────┐
   │         │
  成功      失败
   │         │
   ▼         ▼
┌──────┐  ┌──────────┐
│ 4a.  │  │ 4b.      │
│ 确认 │  │ 退还     │
│ 扣费 │  │ 积分     │
│      │  │          │
│frozen│  │frozen    │
│-=    │  │-= credits│
│spent │  │balance   │
│+=    │  │+= credits│
└──────┘  └──────────┘
```

```typescript
// ================================================================
// 积分事务核心实现
// 使用 D1 事务保证原子性
// ================================================================

type TransactionType = 'earn' | 'spend' | 'freeze' | 'unfreeze' | 'refund'

interface CreditTransaction {
  id: string
  userId: string
  type: TransactionType
  amount: number // 正数 = 收入/退还, 负数 = 消耗/冻结
  balanceAfter: number // 事务后可用余额
  source: string // 'subscription' | 'topup' | 'bonus' | 'invite' | 'ai_call' | 'refund'
  referenceId?: string // 关联的任务/订单 ID
  description?: string
  createdAt: number
}

async function freezeCredits(
  db: D1Database,
  userId: string,
  amount: number,
  referenceId: string,
): Promise<{ success: boolean; error?: string }> {
  // D1 不支持真正的事务, 使用 batch 保证原子性
  const balance = await db
    .prepare('SELECT balance, frozen FROM credit_balances WHERE user_id = ?')
    .bind(userId)
    .first<{ balance: number; frozen: number }>()

  if (!balance || balance.balance < amount) {
    return { success: false, error: 'INSUFFICIENT_CREDITS' }
  }

  const newBalance = balance.balance - amount
  const newFrozen = balance.frozen + amount
  const txId = crypto.randomUUID()

  await db.batch([
    db
      .prepare(
        'UPDATE credit_balances SET balance = ?, frozen = ?, updated_at = ? WHERE user_id = ? AND balance >= ?',
      )
      .bind(newBalance, newFrozen, Date.now(), userId, amount),

    db
      .prepare(
        `INSERT INTO credit_transactions (id, user_id, type, amount, balance_after, source, reference_id, created_at)
       VALUES (?, ?, 'freeze', ?, ?, 'ai_call', ?, ?)`,
      )
      .bind(txId, userId, -amount, newBalance, referenceId, Date.now()),
  ])

  return { success: true }
}

async function confirmSpend(
  db: D1Database,
  userId: string,
  amount: number,
  referenceId: string,
): Promise<void> {
  const txId = crypto.randomUUID()

  await db.batch([
    db
      .prepare(
        'UPDATE credit_balances SET frozen = frozen - ?, total_spent = total_spent + ?, updated_at = ? WHERE user_id = ?',
      )
      .bind(amount, amount, Date.now(), userId),

    db
      .prepare(
        `INSERT INTO credit_transactions (id, user_id, type, amount, balance_after, source, reference_id, created_at)
       VALUES (?, ?, 'spend', ?, (SELECT balance FROM credit_balances WHERE user_id = ?), 'ai_call', ?, ?)`,
      )
      .bind(txId, userId, -amount, userId, referenceId, Date.now()),
  ])
}

async function refundCredits(
  db: D1Database,
  userId: string,
  amount: number,
  referenceId: string,
): Promise<void> {
  const txId = crypto.randomUUID()

  await db.batch([
    db
      .prepare(
        'UPDATE credit_balances SET balance = balance + ?, frozen = frozen - ?, updated_at = ? WHERE user_id = ?',
      )
      .bind(amount, amount, Date.now(), userId),

    db
      .prepare(
        `INSERT INTO credit_transactions (id, user_id, type, amount, balance_after, source, reference_id, created_at)
       VALUES (?, ?, 'refund', ?, (SELECT balance FROM credit_balances WHERE user_id = ?), 'ai_call', ?, ?)`,
      )
      .bind(txId, userId, amount, userId, referenceId, Date.now()),
  ])
}
```

## 四、双模式 AI 执行架构

### 4.1 架构总览

```
                    用户请求 (POST /api/ai/execute)
                              │
                              ▼
                    ┌──────────────────┐
                    │ 读取执行模式      │
                    │ (节点级 > 用户级) │
                    └────────┬─────────┘
                             │
                  ┌──────────┴──────────┐
                  │                     │
            模式一: 积分            模式二: 自带 Key
                  │                     │
                  ▼                     ▼
         ┌────────────────┐    ┌────────────────┐
         │ 验证余额        │    │ 解密用户 Key    │
         │ 冻结积分        │    │ (AES-256-GCM)  │
         └───────┬────────┘    └───────┬────────┘
                 │                     │
                 ▼                     ▼
         ┌────────────────────────────────────┐
         │     AI Provider 统一抽象层          │
         │  ┌──────┐ ┌──────┐ ┌──────┐       │
         │  │OpenR.│ │Gemini│ │Kling │ ...   │
         │  └──────┘ └──────┘ └──────┘       │
         └────────────────┬───────────────────┘
                          │
                     ┌────┴────┐
                    成功      失败
                     │         │
                     ▼         ▼
              确认扣费     退还冻结积分
              (积分模式)   (积分模式)
```

### 4.2 模式一：积分模式

用户使用平台积分，后端使用平台 API Key 代理调用。

```typescript
// ================================================================
// 积分模式执行流程
// ================================================================

async function executeWithCredits(
  ctx: ExecutionContext,
  request: AIExecuteRequest,
): Promise<AIExecuteResponse> {
  const { userId, provider, modelId, input } = request

  // --- Step 1: 查询模型定价 ---
  const pricing = await getModelPricing(ctx.db, provider, modelId)
  if (!pricing) throw new AppError('MODEL_NOT_FOUND', 404)

  // --- Step 2: 冻结积分 ---
  const freezeResult = await freezeCredits(
    ctx.db,
    userId,
    pricing.creditsPerCall,
    request.taskId,
  )
  if (!freezeResult.success) {
    throw new AppError('INSUFFICIENT_CREDITS', 402)
  }

  try {
    // --- Step 3: 使用平台 Key 调用 AI ---
    const aiProvider = getAIProvider(provider)
    const result = await aiProvider.execute({
      modelId,
      input,
      apiKey: ctx.env[`${provider.toUpperCase()}_API_KEY`], // 平台 Key
    })

    // --- Step 4a: 成功 → 确认扣费 ---
    await confirmSpend(ctx.db, userId, pricing.creditsPerCall, request.taskId)

    return result
  } catch (error) {
    // --- Step 4b: 失败 → 退还积分 ---
    await refundCredits(ctx.db, userId, pricing.creditsPerCall, request.taskId)
    throw error
  }
}
```

### 4.3 模式二：自带 Key 模式

用户使用自己的 API Key，不消耗平台积分。后端仍然代理请求（不暴露前端直调）。

```typescript
// ================================================================
// 自带 Key 模式执行流程
// ================================================================

async function executeWithUserKey(
  ctx: ExecutionContext,
  request: AIExecuteRequest,
): Promise<AIExecuteResponse> {
  const { userId, provider, modelId, input } = request

  // --- Step 1: 获取并解密用户 Key ---
  const encryptedKey = await getUserAPIKey(ctx.db, userId, provider)
  if (!encryptedKey) {
    throw new AppError('API_KEY_NOT_CONFIGURED', 400)
  }
  const apiKey = await decryptAPIKey(encryptedKey, ctx.env.ENCRYPTION_KEY)

  // --- Step 2: 使用用户 Key 调用 AI (不扣积分) ---
  const aiProvider = getAIProvider(provider)
  const result = await aiProvider.execute({
    modelId,
    input,
    apiKey,
  })

  // --- Step 3: 记录使用日志 (不扣积分, 仅统计) ---
  await logUsage(ctx.db, userId, provider, modelId, 'user_key')

  return result
}
```

### 4.4 模式选择优先级

```

       │
       ├── 'credits'   → 使用积分模式

              │
              ▼
用户级配置 (user_settings.default_execution_mode)
```

### 4.5 API Key 加密存储方案

使用 AES-256-GCM 对称加密，密钥存储在 Worker 环境变量中。

```typescript
// ================================================================
// API Key 加密/解密
// 使用 Web Crypto API (Cloudflare Workers 原生支持)
// ================================================================

async function encryptAPIKey(plainKey: string, encryptionKey: string): Promise<string> {
  const encoder = new TextEncoder()
  const iv = crypto.getRandomValues(new Uint8Array(12))

  const key = await crypto.subtle.importKey(
    'raw',
    hexToBuffer(encryptionKey),
    { name: 'AES-GCM' },
    false,
    ['encrypt'],
  )

  const encrypted = await crypto.subtle.encrypt(
    { name: 'AES-GCM', iv },
    key,
    encoder.encode(plainKey),
  )

  // 格式: iv(hex):ciphertext(hex)
  return bufferToHex(iv) + ':' + bufferToHex(new Uint8Array(encrypted))
}

async function decryptAPIKey(
  encryptedKey: string,
  encryptionKey: string,
): Promise<string> {
  const [ivHex, ciphertextHex] = encryptedKey.split(':')
  const iv = hexToBuffer(ivHex)
  const ciphertext = hexToBuffer(ciphertextHex)

  const key = await crypto.subtle.importKey(
    'raw',
    hexToBuffer(encryptionKey),
    { name: 'AES-GCM' },
    false,
    ['decrypt'],
  )

  const decrypted = await crypto.subtle.decrypt({ name: 'AES-GCM', iv }, key, ciphertext)

  return new TextDecoder().decode(decrypted)
}
```

## 五、Stripe 集成

### 5.1 Stripe 资源建模

Stripe 资源不直接镜像业务模型，而是作为支付网关存在；业务权益仍以本地表为真相源。

#### Product / Price 规划

1. 套餐 Product
   - `plan_standard`
   - `plan_pro`
   - `plan_ultimate`
2. 套餐 Price
   - 每档至少两种购买模式：`auto_monthly`、`one_time`
   - 每个购买模式下按币种拆成多个 Price
3. 积分包 Product
   - `credits_pack_500`
   - `credits_pack_1200`
   - `credits_pack_3500`
   - `credits_pack_8000`
4. 积分包 Price
   - 每个积分包按支持币种预建 Price

#### Metadata 统一规则

所有 Checkout Session / Subscription / Payment Intent 至少写入：

```ts
{
  userId: string
  actorType: 'clerk'
  orderKind: 'plan_auto_monthly' | 'plan_one_time' | 'credit_pack'
  plan: 'standard' | 'pro' | 'ultimate' | ''
  packageId: string
  currency: string
}
```

### 5.2 Stripe 环境变量与密钥边界

运行时最少需要以下环境变量：

```bash
STRIPE_SECRET_KEY=
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=
STRIPE_WEBHOOK_SECRET=
STRIPE_PRICE_STANDARD_AUTO_MONTHLY_USD=
STRIPE_PRICE_STANDARD_ONE_TIME_USD=
STRIPE_PRICE_PRO_AUTO_MONTHLY_USD=
STRIPE_PRICE_PRO_ONE_TIME_USD=
STRIPE_PRICE_ULTIMATE_AUTO_MONTHLY_USD=
STRIPE_PRICE_ULTIMATE_ONE_TIME_USD=
STRIPE_PRICE_PACK_500_USD=
STRIPE_PRICE_PACK_1200_USD=
STRIPE_PRICE_PACK_3500_USD=
STRIPE_PRICE_PACK_8000_USD=
```

设计原则：

1. 所有 Price ID 只在服务端环境变量中维护，不把业务定价硬编码到客户端。
2. Publishable Key 只用于 Stripe 前端初始化，不参与价格决策。
3. Webhook Secret 只用于 `/api/webhooks/stripe` 签名验证，不与其他密钥复用。

### 5.3 Checkout 选择器

服务端需要一个明确的 Price 解析层：

```ts
type CheckoutSelection =
  | { kind: 'plan'; plan: 'standard' | 'pro' | 'ultimate'; purchaseMode: 'auto_monthly' | 'one_time'; currency: CurrencyCode }
  | { kind: 'credit_pack'; packageId: CreditPackageId; currency: CurrencyCode }
```

```ts
function resolveStripePriceId(selection: CheckoutSelection): string
```

约束：

1. `free` 不进入 Stripe Checkout。
2. 自动月付只能映射到 `subscription` 模式 Price。
3. 一次性套餐与积分包只能映射到 `payment` 模式 Price。
4. `currency` 不命中时回退站点默认币种，并记录日志。

### 5.4 Customer 与本地账户映射

1. 每个正式登录账户最多对应一个 Stripe Customer。
2. `stripe_customer_id` 存在本地 `subscriptions` 或专门的 `billing_customers` 表中。
3. 匿名用户不创建 Stripe Customer，支付入口要求登录后再发起结算。
4. 当 Clerk 用户资料更新时，不主动回写 Stripe；Stripe 只需要最小账单身份。

### 5.5 Checkout 创建流程 (Stripe Checkout)

```
用户点击 "购买套餐" 或 "购买积分包"
       │
       ▼
前端: POST /api/billing/checkout
       │
       ▼
后端:
  1. 获取或创建 Stripe Customer (关联 Clerk userId)
  2. 根据 plan + purchaseMode + currency 选择 Stripe Price
  3. 创建 Stripe Checkout Session
     - 套餐自动月付: mode = 'subscription'
     - 套餐一次性购买: mode = 'payment'
     - 积分包购买: mode = 'payment'
     - success_url: '/billing?success=true'
     - cancel_url: '/pricing'
     - metadata: { userId, plan, purchaseMode, currency }
  4. 返回 session.url
       │
       ▼
前端: window.location.href = session.url
       │
       ▼
Stripe Checkout 页面 (Stripe 托管)
       │
       ▼
支付成功 → 重定向到 /billing?success=true
       │
       ▼ (异步)
Stripe Webhook → checkout.session.completed
       │
       ▼
后端:
  1. 识别订单类型 (自动月付套餐 / 一次性套餐 / 积分包)
  2. 写入 subscriptions 或 orders 表
  3. 发放对应积分与存储权益
```

```typescript
// ================================================================
// 创建 Stripe Checkout Session
// ================================================================

import Stripe from 'stripe'

async function createCheckoutSession(
  stripe: Stripe,
  db: D1Database,
  userId: string,
  plan: 'free' | 'standard' | 'pro' | 'ultimate',
  purchaseMode: 'auto_monthly' | 'one_time' | 'credit_pack',
  currency: 'usd' | 'eur' | 'gbp' | 'cny',
  creditPackageId?: string,
): Promise<string> {
  // --- 获取或创建 Stripe Customer ---
  let customer = await getStripeCustomer(db, userId)
  if (!customer) {
    const stripeCustomer = await stripe.customers.create({
      metadata: { userId },
    })
    await saveStripeCustomer(db, userId, stripeCustomer.id)
    customer = stripeCustomer.id
  }

  // --- 创建 Checkout Session ---
  const priceId = resolveStripePriceId({
    plan,
    purchaseMode,
    currency,
    creditPackageId,
  })
  const isSubscription = purchaseMode === 'auto_monthly'
  const session = await stripe.checkout.sessions.create({
    customer,
    mode: isSubscription ? 'subscription' : 'payment',
    line_items: [{ price: priceId, quantity: 1 }],
    success_url: `${process.env.NEXT_PUBLIC_URL}/billing?success=true`,
    cancel_url: `${process.env.NEXT_PUBLIC_URL}/pricing`,
    metadata: { userId, plan, purchaseMode, currency, creditPackageId: creditPackageId ?? '' },
    ...(isSubscription
      ? {
          subscription_data: {
            metadata: { userId, plan, purchaseMode },
          },
        }
      : {}),
  })

  return session.url!
}
```

### 5.6 Stripe Webhook 事件处理

```typescript
// ================================================================
// Stripe Webhook 处理器
// 所有积分充值、订阅状态变更都通过 Webhook 驱动
// ================================================================

type StripeWebhookHandler = (event: Stripe.Event, db: D1Database) => Promise<void>

const webhookHandlers: Record<string, StripeWebhookHandler> = {
  // --- 订阅首次创建成功 ---
  'checkout.session.completed': async (event, db) => {
    const session = event.data.object as Stripe.Checkout.Session
    const { userId, plan, purchaseMode } = session.metadata!

    if (session.mode === 'subscription') {
      // 自动月付套餐: 创建订阅记录 + 充值积分 + 开通存储权益
      const subscription = await stripe.subscriptions.retrieve(
        session.subscription as string,
      )
      await createSubscription(db, userId, plan, subscription)
      await addCredits(db, userId, PLAN_CREDITS[plan], 'subscription', subscription.id)
      await syncStorageQuota(db, userId, plan)
    } else if (session.mode === 'payment') {
      if (purchaseMode === 'credit_pack') {
        // 积分包购买
        const { creditAmount } = session.metadata!
        await addCredits(db, userId, parseInt(creditAmount), 'topup', session.id)
      } else {
        // 一次性套餐
        await grantOneTimePlan(db, userId, plan, session.id)
        await addCredits(db, userId, PLAN_CREDITS[plan], 'one_time_plan', session.id)
        await syncStorageQuota(db, userId, plan)
      }
    }
  },

  // --- 自动月付套餐续费成功 ---
  'invoice.paid': async (event, db) => {
    const invoice = event.data.object as Stripe.Invoice
    if (invoice.billing_reason === 'subscription_cycle') {
      const subscription = await stripe.subscriptions.retrieve(
        invoice.subscription as string,
      )
      const { userId, plan } = subscription.metadata

      // 重置当期积分 (不是累加, 是重置)
      await resetMonthlyCredits(db, userId, PLAN_CREDITS[plan])
      await syncStorageQuota(db, userId, plan)
    }
  },

  // --- 订阅取消 ---
  'customer.subscription.deleted': async (event, db) => {
    const subscription = event.data.object as Stripe.Subscription
    const { userId } = subscription.metadata

    await updateSubscription(db, userId, {
      status: 'canceled',
      plan: 'free',
      monthlyCredits: PLAN_CREDITS.free,
    })

    // 重置为免费套餐权益
    await resetMonthlyCredits(db, userId, PLAN_CREDITS.free)
    await syncStorageQuota(db, userId, 'free')
  },

  // --- 订阅变更 (升级/降级) ---
  'customer.subscription.updated': async (event, db) => {
    const subscription = event.data.object as Stripe.Subscription
    const previous = event.data.previous_attributes as Partial<Stripe.Subscription>
    const { userId, plan } = subscription.metadata

    // 只在套餐实际变更时处理
    if (previous.items) {
      await updateSubscription(db, userId, {
        plan,
        monthlyCredits: PLAN_CREDITS[plan],
        status: subscription.status === 'active' ? 'active' : 'past_due',
      })
      await syncStorageQuota(db, userId, plan)
    }
  },

  // --- 支付失败 ---
  'invoice.payment_failed': async (event, db) => {
    const invoice = event.data.object as Stripe.Invoice
    const subscription = await stripe.subscriptions.retrieve(
      invoice.subscription as string,
    )
    const { userId } = subscription.metadata

    await updateSubscription(db, userId, { status: 'past_due' })
    // 可选: 发送通知邮件
  },
}

// --- Webhook 入口 ---
async function handleStripeWebhook(
  request: Request,
  env: Env,
  db: D1Database,
): Promise<Response> {
  const signature = request.headers.get('stripe-signature')!
  const body = await request.text()

  let event: Stripe.Event
  try {
    event = stripe.webhooks.constructEvent(body, signature, env.STRIPE_WEBHOOK_SECRET)
  } catch {
    return new Response('Invalid signature', { status: 400 })
  }

  const handler = webhookHandlers[event.type]
  if (handler) {
    await handler(event, db)
  }

  return new Response('OK', { status: 200 })
}
```

### 5.7 Webhook 幂等与落账顺序

Webhook 必须把“验签、幂等、落账、权益同步”拆成稳定顺序：

1. 验证 `stripe-signature`
2. 查询 `processed_stripe_events`
3. 未处理才进入业务逻辑
4. 先写原始事件审计，再写本地账本
5. 本地账本成功后，标记事件已处理

推荐表：

```sql
CREATE TABLE processed_stripe_events (
  event_id TEXT PRIMARY KEY,
  event_type TEXT NOT NULL,
  processed_at INTEGER NOT NULL
);
```

落账优先级：

1. `checkout.session.completed`
2. `invoice.paid`
3. `customer.subscription.updated`
4. `customer.subscription.deleted`
5. `invoice.payment_failed`

```
GET    /api/billing/subscription
  → 200 { plan, purchaseMode, status, billingPeriod, currentPeriodEnd, monthlyCredits, storageGB, cancelAtPeriodEnd }

POST   /api/billing/checkout
  Body: { plan: 'free'|'standard'|'pro'|'ultimate', purchaseMode: 'auto_monthly'|'one_time'|'credit_pack', currency?: string, creditPackageId?: string }
  → 200 { checkoutUrl: string }

POST   /api/billing/portal
  → 200 { portalUrl: string }

POST   /api/billing/cancel
  → 200 { canceledAt: number }
```

Checkout 保护规则：

1. 未登录用户访问 `/api/billing/checkout` 直接返回 401，并在前端回跳 `/sign-in?redirect_url=/pricing`。
2. `free` 不允许直接发起 Checkout。
3. 参数中的 `currency` 只能是白名单币种，不能直接信任客户端任意值。
4. 价格 ID 只能由服务端 `resolveStripePriceId()` 选择。

### 9.2 并发扣费的原子性

```typescript
// ================================================================
// D1 batch 保证积分操作的原子性
// 所有读-判断-写操作通过 batch 执行
// ================================================================

// 关键: 使用 WHERE balance >= amount 的条件更新
// 如果余额不足, UPDATE 影响行数为 0, 整个 batch 无效果

async function atomicFreezeCredits(
  db: D1Database,
  userId: string,
  amount: number,
): Promise<boolean> {
  const results = await db.batch([
    // 条件更新: 只有余额充足时才会生效
    db
      .prepare(
        `UPDATE credit_balances
       SET monthly_balance = CASE
         WHEN monthly_balance >= ? THEN monthly_balance - ?
         ELSE monthly_balance
       END,
       permanent_balance = CASE
         WHEN monthly_balance < ? THEN permanent_balance - (? - monthly_balance)
         ELSE permanent_balance
       END,
       frozen = frozen + ?,
       updated_at = ?
       WHERE user_id = ?
       AND (monthly_balance + permanent_balance) >= ?`,
      )
      .bind(amount, amount, amount, amount, amount, Date.now(), userId, amount),
  ])

  // 检查是否真正更新了行
  return (results[0] as D1Result).meta.changes > 0
}
```

```typescript
// Stripe Webhook 安全验证
// 必须验证签名, 防止伪造 Webhook

async function verifyStripeSignature(
  body: string,
  signature: string,
  secret: string,
): Promise<Stripe.Event> {
  try {
    return stripe.webhooks.constructEvent(body, signature, secret)
  } catch (err) {
    throw new AppError('INVALID_WEBHOOK_SIGNATURE', 400)
  }

// 幂等性保证: 通过 event.id 去重
// Stripe 可能重复发送同一事件
async function isEventProcessed(db: D1Database, eventId: string): Promise<boolean> {
  const row = await db
    .prepare('SELECT id FROM processed_stripe_events WHERE event_id = ?')
    .bind(eventId)
    .first()
  return !!row
}

async function markEventProcessed(db: D1Database, eventId: string): Promise<void> {
  await db
    .prepare(
      'INSERT OR IGNORE INTO processed_stripe_events (event_id, processed_at) VALUES (?, ?)',
    )
    .bind(eventId, Date.now())
    .run()
}
```
