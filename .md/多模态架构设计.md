## 一、概述

### 1.1 设计目标

实现节点间多种数据类型的传递,支持文本、图片、视频、音频、文件等多模态内容在工作流中流转。

### 1.2 核心价值

1. **类型安全**: 编译期验证连线是否合法,避免运行时错误
2. **可扩展性**: 新增数据类型只需修改 PortType 常量对象
3. **单一数据源**: 所有类型定义集中在 `types/multimodal.ts`
4. **兼容性验证**: 自动检查输出端口与输入端口的类型兼容性

## 二、端口类型系统

### 2.1 PortType 常量对象

使用 `as const` 对象代替 enum,兼容 TypeScript `erasableSyntaxOnly` 模式:

```typescript
export const PortType = {
  TEXT: 'text', // 纯文本
  IMAGE: 'image', // 图片 (URL 或 base64 data URL)
  VIDEO: 'video', // 视频 (URL 或 base64 data URL)
  AUDIO: 'audio', // 音频 (URL 或 base64 data URL)
  FILE: 'file', // 通用文件 (包含元数据)
  ANY: 'any', // 任意类型 (通配符)
} as const

export type PortType = (typeof PortType)[keyof typeof PortType]
```

### 2.2 TypedPort 接口

定义端口的完整类型信息:

```typescript
export interface TypedPort {
  id: string // 端口 ID
  label: string // 显示标签
  type: PortType // 输出类型
  accepts?: PortType[] // 接受的输入类型列表 (仅输入端口)
}
```

**示例**:

```typescript
// LLM 节点的输入端口 (接受文本和图片)
const inputPort: TypedPort = {
  id: 'input',
  label: '输入',
  type: PortType.TEXT,
  accepts: [PortType.TEXT, PortType.IMAGE],
}

// 图片生成节点的输出端口
const outputPort: TypedPort = {
  id: 'output',
  label: '输出',
  type: PortType.IMAGE,
}
```

## 三、多模态消息结构

export interface MultiModalMessage {

## 四、端口类型兼容性验证

### 4.1 兼容性规则

```typescript
/**
 * 检查输出端口类型是否与输入端口兼容
 *
 * @param outputType - 输出端口类型
 * @param acceptedTypes - 输入端口接受的类型列表
 * @returns 是否兼容
 */
export function isPortTypeCompatible(
  outputType: PortType,
  acceptedTypes?: PortType[],
): boolean
```

**判断逻辑**:

1. 输入端口未指定 `accepts`,默认接受所有类型 → ✅ 兼容
2. 输入端口 `accepts` 包含 `PortType.ANY` → ✅ 兼容
3. 输出端口类型是 `PortType.ANY` → ✅ 兼容
4. 输出类型在 `accepts` 列表中 → ✅ 兼容
5. 其他情况 → ❌ 不兼容

### 4.2 使用示例

```typescript
// 场景 1: LLM 节点输出 TEXT,Display 节点接受 ANY
isPortTypeCompatible(PortType.TEXT, [PortType.ANY]) // ✅ true

// 场景 2: ImageGen 节点输出 IMAGE,VideoGen 节点仅接受 VIDEO
isPortTypeCompatible(PortType.IMAGE, [PortType.VIDEO]) // ❌ false

// 场景 3: FileUpload 节点输出 FILE,LLM 节点接受 TEXT/IMAGE/FILE
isPortTypeCompatible(PortType.FILE, [PortType.TEXT, PortType.IMAGE, PortType.FILE]) // ✅ true
```

### 4.3 Canvas 集成

在 `Canvas.tsx` 的 `isValidConnection` 回调中验证:

```typescript
const isValidConnection = useCallback(
  (connection: Connection) => {
    const sourceNode = nodes.find((n) => n.id === connection.source)
    const targetNode = nodes.find((n) => n.id === connection.target)

    // 查询端口类型
    const sourcePortType = getPortType(sourceNode.type, connection.sourceHandle!)
    const targetPortAccepts = getPortAccepts(targetNode.type, connection.targetHandle!)

    // 兼容性验证
    const compatible = isPortTypeCompatible(sourcePortType, targetPortAccepts)

    if (!compatible) {
      console.warn(`[Connection Rejected] Port type mismatch`)
    }

    return compatible
  },
  [nodes],
)
```

## 五、节点执行器支持

### 5.1 WorkflowExecutor 改造

支持 `MultiModalMessage` 作为节点间传递的数据格式:

```typescript
// 收集上游节点的多模态输出
const collectInputs = (nodeId: string): MultiModalMessage => {
  const incomingEdges = edges.filter((e) => e.target === nodeId)
  const messages: MultiModalMessage[] = []

  for (const edge of incomingEdges) {
    const sourceOutput = this.nodeOutputs.get(edge.source)
    if (sourceOutput) {
      messages.push(sourceOutput as MultiModalMessage)
    }
  }

  return mergeMessages(...messages)
}
```

### 5.2 节点执行器签名

所有节点执行器统一签名:

```typescript
export type NodeExecutor<T extends BaseNodeData = BaseNodeData> = (
  node: Node<T>,
  inputs: Record<string, unknown>, // 包含 MultiModalMessage
  context: ExecutorContext,
) => Promise<unknown>
```

function MultiModalPreview({ data }: { data: BaseNodeData }) {
  const input = data.input as MultiModalMessage | undefined

  if (!input || isMessageEmpty(input)) return null

  return (
    <div className="multimodal-preview">
      {input.text && <TextPreview text={input.text} />}
      {input.images && <ImageGallery images={input.images} />}
      {input.videos && <VideoPlayer videos={input.videos} />}

## 八、类型系统迁移

### 8.1 旧版 vs 新版

| 特性         | 旧版 (edge.ts)                             | 新版 (multimodal.ts)                     |
| ------------ | ------------------------------------------ | ---------------------------------------- |
| 类型定义     | `type PortType = 'text' \| 'image' \| ...` | `const PortType = { TEXT: 'text', ... }` |
| 兼容性检查   | `PORT_TYPE_COMPATIBILITY` 常量             | `isPortTypeCompatible()` 函数            |
| 端口定义     | 字符串                                     | `TypedPort` 接口                         |
| accepts 支持 | ❌ 无                                      | ✅ 有                                    |
| 消息结构     | `string`                                   | `MultiModalMessage`                      |

### 8.2 迁移步骤

1. ✅ 删除 `edge.ts` 中的旧版 `isPortTypeCompatible` 和 `PORT_TYPE_COMPATIBILITY`
2. ✅ 删除 `node.ts` 中的旧版 `type PortType`
3. ✅ 将 `enum PortType` 改为 `as const` 对象 (兼容 erasableSyntaxOnly)
4. ✅ 所有引用统一使用 `import { PortType } from '@/types'`

## 九、架构决策记录 (ADR)

### 9.1 为何使用 `as const` 对象而非 enum?

**背景**: TypeScript 5.5+ 引入 `erasableSyntaxOnly` 模式,禁止 enum 在运行时使用。

**决策**: 使用 `as const` 对象 + 类型推导。

**优势**:

- ✅ 兼容 `erasableSyntaxOnly` 模式
- ✅ 保持与 enum 相同的使用方式 (`PortType.TEXT`)
- ✅ 生成更小的编译产物 (无 enum 辅助函数)
- ✅ 类型安全不变

### 9.2 为何 MultiModalMessage 所有字段都是可选的?

**背景**: 不同节点输出的数据类型不同。

**决策**: 所有字段 optional,通过 `getMessageContentType()` 判断主要类型。

**优势**:

- ✅ 灵活组合 (文本 + 图片,或单独图片)
- ✅ 避免冗余字段
- ✅ 向后兼容 (新增字段不破坏旧代码)

### 9.3 为何端口类型兼容性在 Canvas 层验证而非 Executor?

**背景**: 可在连线时拒绝,或执行时报错。

**决策**: 在 Canvas 的 `isValidConnection` 中验证。

**优势**:

- ✅ 即时反馈,用户体验好
- ✅ 避免无效工作流保存
- ✅ 减少执行时错误

### 10.2 新增数据类型

添加新类型只需 3 步:

1. 在 `PortType` 常量对象中添加新键值对
2. 在 `MultiModalMessage` 中添加新字段
3. 创建对应的工具函数 (如 `createDocumentMessage()`)

**示例**: 添加 `DOCUMENT` 类型

```typescript
export const PortType = {
  // ... 现有类型
  DOCUMENT: 'document',
} as const

export interface MultiModalMessage {
  // ... 现有字段
  documents?: FileMetadata[]
}

export function createDocumentMessage(doc: FileMetadata): MultiModalMessage {
  return { documents: [doc] }
}
```
