## 技术基础设施

### 1. robots.txt 配置

**作用**: 控制搜索引擎爬虫的访问权限

**文件位置**: `{项目根目录}/public/robots.txt` 或 `static/robots.txt`

**通用模板**:
```txt
# Allow all major search engines
User-agent: Googlebot
User-agent: Bingbot
User-agent: Baiduspider
User-agent: Yandex
Allow: /

# Block suspicious crawlers
User-agent: *
Disallow: /api/
Disallow: /admin/
Disallow: /_auth/
Disallow: /_internal/

# Sitemap location (替换为你的域名)
Sitemap: https://your-domain.com/sitemap.xml

# Crawl delay to protect server
Crawl-delay: 1

# Disallow URL parameters to avoid duplicate content
Disallow: /*?*
Allow: /*?lang=*
Allow: /*?page=*
```

**配置说明**:

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `User-agent: Googlebot` | 显式允许主流搜索引擎 | Google、Bing、百度、Yandex |
| `Disallow: /api/` | 禁止爬取 API 路径 | 避免浪费爬虫配额 |
| `Disallow: /*?*` | 禁止带查询参数的 URL | 避免重复内容索引 |
| `Allow: /*?lang=*` | 允许特定参数 | 多语言、分页等必要参数 |
| `Crawl-delay: 1` | 设置爬取延迟（秒） | 保护服务器压力 |

**验证方法**:
```bash
# 部署后访问
curl https://your-domain.com/robots.txt

# 使用 Google Search Console
# Settings → robots.txt Tester
```

**常见错误**:
- ❌ 不小心禁止了重要页面 (`Disallow: /`)
- ❌ 忘记添加 Sitemap 引用
- ❌ 过度限制导致页面无法被索引

### 2. Sitemap 系统

**作用**: 告诉搜索引擎网站有哪些页面需要索引

#### 2.1 XML Sitemap 结构

**基础模板**:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"
        xmlns:xhtml="http://www.w3.org/1999/xhtml">

  <!-- 首页 -->
  <url>
    <loc>https://your-domain.com/</loc>
    <lastmod>2025-12-30</lastmod>
    <changefreq>daily</changefreq>
    <priority>1.0</priority>
  </url>

  <!-- 分类页 -->
  <url>
    <loc>https://your-domain.com/category/technology</loc>
    <lastmod>2025-12-25</lastmod>
    <changefreq>weekly</changefreq>
    <priority>0.8</priority>
  </url>

  <!-- 内容详情页 -->
  <url>
    <loc>https://your-domain.com/blog/seo-guide</loc>
    <lastmod>2025-12-20</lastmod>
    <changefreq>weekly</changefreq>
    <priority>0.7</priority>
  </url>
</urlset>
```

**字段说明**:

| 字段 | 必需 | 说明 | 建议值 |
|------|------|------|--------|
| `<loc>` | ✅ | 页面完整 URL | 必须以 http:// 或 https:// 开头 |
| `<lastmod>` | 推荐 | 最后修改时间 | YYYY-MM-DD 格式 |
| `<changefreq>` | 可选 | 更新频率 | daily/weekly/monthly |
| `<priority>` | 可选 | 优先级 (0.0-1.0) | 首页 1.0，其他 0.5-0.9 |

---

#### 2.2 Priority 设置规则

| 页面类型 | Priority | 理由 |
|---------|----------|------|
| 首页 | 1.0 | 最重要入口 |
| 主要分类页 | 0.9 | 高价值聚合页 |
| 次要分类页 | 0.8 | 一般聚合页 |
| 内容详情页 | 0.7 | 核心内容 |
| 标签页 | 0.6 | 辅助聚合 |
| 静态页面 | 0.5 | 关于、联系等 |

---

#### 2.3 Changefreq 设置规则

| 页面类型 | Changefreq | 理由 |
|---------|------------|------|
| 首页 | daily | 内容频繁更新 |
| 博客列表 | daily | 新文章发布 |
| 分类页 | weekly | 定期有新内容 |
| 详情页 | weekly | 偶尔修订 |
| 静态页面 | monthly | 很少变化 |

---

#### 2.4 动态 Sitemap 实现

**方案A: Node.js/Express**
```javascript
// routes/sitemap.js
const { SitemapStream, streamToPromise } = require('sitemap');

app.get('/sitemap.xml', async (req, res) => {
  const sitemap = new SitemapStream({
    hostname: 'https://your-domain.com'
  });

  // 添加首页
  sitemap.write({
    url: '/',
    changefreq: 'daily',
    priority: 1.0
  });

  // 从数据库读取内容页
  const posts = await db.query('SELECT slug, updated_at FROM posts');
  posts.forEach(post => {
    sitemap.write({
      url: `/blog/${post.slug}`,
      lastmod: post.updated_at,
      changefreq: 'weekly',
      priority: 0.7
    });
  });

  sitemap.end();
  const xml = await streamToPromise(sitemap);
  res.header('Content-Type', 'application/xml');
  res.send(xml.toString());
});
```

**方案B: Next.js**
```javascript
// pages/sitemap.xml.js
export default function Sitemap() {}

export async function getServerSideProps({ res }) {
  const posts = await getAllPosts();

  const sitemap = `<?xml version="1.0" encoding="UTF-8"?>
    <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
      <url>
        <loc>https://your-domain.com</loc>
        <changefreq>daily</changefreq>
        <priority>1.0</priority>
      </url>
      ${posts.map(post => `
        <url>
          <loc>https://your-domain.com/blog/${post.slug}</loc>
          <lastmod>${post.updatedAt}</lastmod>
          <changefreq>weekly</changefreq>
          <priority>0.7</priority>
        </url>
      `).join('')}
    </urlset>
  `;

  res.setHeader('Content-Type', 'application/xml');
  res.write(sitemap);
  res.end();

  return { props: {} };
}
```

**方案C: 静态生成（构建时）**
```javascript
// scripts/generate-sitemap.js
const fs = require('fs');
const posts = require('./content/posts.json');

const sitemap = `<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  ${posts.map(post => `
    <url>
      <loc>https://your-domain.com/blog/${post.slug}</loc>
      <lastmod>${post.updatedAt}</lastmod>
      <changefreq>weekly</changefreq>
      <priority>0.7</priority>
    </url>
  `).join('')}
</urlset>
`;

fs.writeFileSync('public/sitemap.xml', sitemap);
```

**实现要点**:
- ✅ 单个 sitemap 不超过 50,000 条 URL
- ✅ 文件大小不超过 50MB（未压缩）
- ✅ 使用缓存（24小时）减轻服务器压力
- ✅ 超过限制时使用 sitemap index

### 3. Canonical 规则

**作用**: 避免重复内容问题，告诉搜索引擎哪个是规范 URL

#### 3.1 基础实现

**HTML 中添加**:
```html
<head>
  <link rel="canonical" href="https://your-domain.com/blog/seo-guide" />
</head>
```

#### 3.2 动态更新（SPA 项目）

**React 示例**:
```javascript
// utils/seo.js
export function updateCanonical(url) {
  let canonical = document.querySelector('link[rel="canonical"]');
  if (!canonical) {
    canonical = document.createElement('link');
    canonical.setAttribute('rel', 'canonical');
    document.head.appendChild(canonical);
  }
  canonical.setAttribute('href', url);
}

// 在组件中使用
import { useEffect } from 'react';
import { useLocation } from 'react-router-dom';
import { updateCanonical } from './utils/seo';

function BlogPost() {
  const location = useLocation();

  useEffect(() => {
    updateCanonical(`https://your-domain.com${location.pathname}`);
  }, [location]);

  return <article>...</article>;
}
```

#### 3.3 常见场景

**场景1: 分页内容**
```html
<!-- 第1页（规范 URL） -->
<link rel="canonical" href="https://your-domain.com/blog" />

<!-- 第2页（指向第1页） -->
<link rel="canonical" href="https://your-domain.com/blog?page=2" />
```

**场景2: 多个 URL 指向同一内容**
```html
<!-- /product/123 和 /products/my-product 都指向同一产品 -->
<link rel="canonical" href="https://your-domain.com/product/123" />
```

**场景3: HTTP vs HTTPS**
```html
<!-- HTTP 版本指向 HTTPS -->
<link rel="canonical" href="https://your-domain.com/page" />
```

**关键规则**:
- ✅ 每个页面必须有 canonical 标签
- ✅ 自引用（页面指向自己）是最佳实践
- ✅ URL 末尾统一处理（带/或不带/）
- ✅ 使用绝对 URL（带协议和域名）

### 4. 重定向策略

#### 4.1 301 重定向（永久）

**用途**: 旧 URL 永久迁移到新 URL

**Express 示例**:
```javascript
// 移除尾部斜杠
app.use((req, res, next) => {
  const { path } = req;
  if (path !== '/' && path.endsWith('/')) {
    return res.redirect(301, path.slice(0, -1));
  }
  next();
});

// 自定义重定向规则
const redirectRules = {
  '/old-blog-post': '/new-blog-post',
  '/products': '/shop',
};

app.use((req, res, next) => {
  if (redirectRules[req.path]) {
    return res.redirect(301, redirectRules[req.path]);
  }
  next();
});
```

**Nginx 示例**:
```nginx
# 移除尾部斜杠
rewrite ^/(.*)/$ /$1 permanent;

# 旧域名重定向到新域名
server {
  server_name old-domain.com;
  return 301 https://new-domain.com$request_uri;
}

# 单个页面重定向
location = /old-page {
  return 301 /new-page;
}
```

**常见场景**:
```
# 移除尾部斜杠
/blog/seo-guide/ → /blog/seo-guide (301)

# 处理双斜杠
/blog//seo-guide → /blog/seo-guide (301)

# 旧 URL 迁移
/old-blog/123 → /blog/seo-guide (301)

# 域名迁移
http://old-domain.com/page → https://new-domain.com/page (301)
```

#### 4.2 404 页面优化

**SEO 友好的 404 页面要素**:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Page Not Found - Your Site</title>

  <!-- 重要：告诉搜索引擎不要索引 404 页面 -->
  <meta name="robots" content="noindex, nofollow">

  <meta name="description" content="The page you're looking for doesn't exist.">
</head>
<body>
  <h1>404 - Page Not Found</h1>
  <p>Sorry, the page you're looking for doesn't exist.</p>

  <!-- 搜索框 -->
  <form action="/search">
    <input type="text" name="q" placeholder="Search...">
    <button type="submit">Search</button>
  </form>

  <!-- 热门内容链接 -->
  <section>
    <h2>Popular Pages</h2>
    <ul>
      <li><a href="/">Home</a></li>
      <li><a href="/blog">Blog</a></li>
      <li><a href="/products">Products</a></li>
    </ul>
  </section>

  <!-- 返回首页按钮 -->
  <a href="/" class="btn">Go to Homepage</a>
</body>
</html>
```

**服务端配置**（确保返回 HTTP 404 状态码）:
```javascript
// Express
app.use((req, res) => {
  res.status(404).render('404');

  return <h1>404 - Page Not Found</h1>
}
```

## 多语言 SEO 策略

### 1. URL 结构选择

**方案对比**:

| 方案 | URL 示例 | 优点 | 缺点 | 推荐度 |
|------|---------|------|------|--------|
| **子目录** | `/en/`, `/zh/` | 易管理、权重集中、成本低 | - | ⭐⭐⭐⭐⭐ |
| **子域名** | `en.domain.com` | 可独立管理 | 权重分散、需多个SSL | ⭐⭐⭐ |
| **独立域名** | `domain.com`, `domain.cn` | 地域性强 | 成本高、权重分散 | ⭐⭐ |
| **参数** | `?lang=en` | 实现简单 | SEO 不友好 | ⭐ |

**推荐**: 使用子目录方案

### 2. hreflang 标签实现

**作用**: 告诉搜索引擎页面的语言版本关系

**HTML 实现**:
```html
<head>
  <!-- 英文版（自引用） -->
  <link rel="alternate" hreflang="en"
        href="https://your-domain.com/en/page" />

  <!-- 中文版 -->
  <link rel="alternate" hreflang="zh"
        href="https://your-domain.com/zh/page" />
  <link rel="alternate" hreflang="zh-CN"
        href="https://your-domain.com/zh/page" />

  <!-- 默认版本 -->
  <link rel="alternate" hreflang="x-default"
        href="https://your-domain.com/en/page" />
</head>
```

**动态生成（React 示例）**:
```javascript
function updateHreflangTags(pathname, currentLang) {
  // 移除现有标签
  document.querySelectorAll('link[hreflang]').forEach(el => el.remove());

  const languages = [
    { code: 'en', url: `/en${pathname}` },
    { code: 'zh', url: `/zh${pathname}` },
    { code: 'zh-CN', url: `/zh${pathname}` },
    { code: 'x-default', url: `/en${pathname}` }
  ];

  languages.forEach(({ code, url }) => {
    const link = document.createElement('link');
    link.rel = 'alternate';
    link.hreflang = code;
    link.href = `https://your-domain.com${url}`;
    document.head.appendChild(link);
  });
}
```

**关键规则**:
- ✅ 每个语言版本都要包含指向所有版本的 hreflang
- ✅ 必须包含自引用（英文页面也要有 `hreflang="en"`）
- ✅ 使用 `x-default` 指定默认语言
- ✅ 所有 hreflang URL 必须返回 200 OK（不能是 302）

---

### 3. 地理重定向

**根据用户地理位置自动重定向到相应语言版本**

**Cloudflare Workers 示例**:
```javascript
export default {
  async fetch(request) {
    const url = new URL(request.url);
    const country = request.headers.get('CF-IPCountry') || '';

    // 地理位置到语言的映射
    const countryLangMap = {
      'CN': 'zh', 'HK': 'zh', 'TW': 'zh', 'MO': 'zh',
      'US': 'en', 'GB': 'en', 'CA': 'en', 'AU': 'en',
      'FR': 'fr', 'DE': 'de', 'ES': 'es'
    };

    const targetLang = countryLangMap[country] || 'en';

    // 仅对根路径重定向
    if (url.pathname === '/' || url.pathname === '') {
      return Response.redirect(
        new URL(`/${targetLang}`, url.origin),
        302 // 使用 302 临时重定向，允许用户手动切换
      );
    }

    return fetch(request);
  }
};
```

**注意事项**:
- 使用 302 临时重定向（不是 301）
- 保留用户的语言选择（Cookie）
- 不要重定向 sitemap、API 等路径

### 3. Meta Keywords 实现

```javascript
function generateMetaKeywords(content) {
  const allKeywords = [
    ...(content.keywords || []),                    // 主关键词
    ...(content.long_tail_keywords || []).slice(0, 5), // 长尾词（前5个）

  metaKeywords.setAttribute('content', generateMetaKeywords(content));

function updateDetailPageSEO(content) {
  const mainKeyword = content.keywords?.[0] || content.title;
  const tags = content.tags?.slice(0, 3).map(t => t.name).join(', ');

  // Title（包含主关键词）

  updateMetaTag('description', description.slice(0, 160));

  // Meta Keywords
  updateMetaTag('keywords', generateMetaKeywords(content));

  // Canonical
  updateCanonical(`https://your-domain.com${window.location.pathname}`);

  // Open Graph
  updateMetaTag('og:title', title, 'property');
  updateMetaTag('og:description', description, 'property');
  updateMetaTag('og:image', content.image, 'property');
}

function updateMetaTag(name, content, attribute = 'name') {
  let meta = document.querySelector(`meta[${attribute}="${name}"]`);
  if (!meta) {
    meta = document.createElement('meta');
    meta.setAttribute(attribute, name);
    document.head.appendChild(meta);
  }
  meta.setAttribute('content', content);
}

## 问题排查

### 1. 页面没有被索引

**检查步骤**:
```bash
# 1. 检查 robots.txt
curl https://your-domain.com/robots.txt

# 2. 检查是否有 noindex 标签
curl -s https://your-domain.com/page | grep "noindex"

# 3. 检查 HTTP 状态码
curl -I https://your-domain.com/page
```

**常见原因**:
- ❌ robots.txt 阻止爬取
- ❌ 页面有 `<meta name="robots" content="noindex">`
- ❌ 页面返回 404 或 500 错误
- ❌ 页面是孤岛（无内部链接）
- ❌ Sitemap 未提交或不包含该页面

**解决方法**:
1. 在 GSC 中使用"URL 检查"工具
2. 手动请求编入索引
3. 确保页面有内部链接指向
4. 检查并修复技术问题

---

### 2. 关键词排名下降

4. 检查页面加载速度

**优化方法**:
1. 更新和丰富页面内容
2. 增加内部链接
3. 获取更多外部链接
4. 改善用户体验（降低跳出率）

### 4. hreflang 错误

**常见错误**:

**错误1: 缺少自引用**
```html
<!-- ❌ 错误：英文页面没有 hreflang="en" -->
<link rel="alternate" hreflang="zh" href=".../zh/page" />

<!-- ✅ 正确：包含自引用 -->
<link rel="alternate" hreflang="en" href=".../en/page" />
<link rel="alternate" hreflang="zh" href=".../zh/page" />
```

**错误2: Missing return link**
```
问题：英文页面有指向中文的 hreflang，但中文页面没有指向英文的
解决：确保所有语言版本互相链接
```

**错误3: URL 返回 302**
```
问题：hreflang 指向的 URL 经过重定向
解决：hreflang 必须指向最终 URL
```

**验证工具**:
- https://technicalseo.com/tools/hreflang/
- https://www.aleydasolis.com/hreflang-tag-generator-tool/

### 9.1 地理重定向实现（Cloudflare Workers）

**文件位置**：`backend/src/middleware/geoRedirect.ts`

**核心逻辑**：
```typescript
export async function geoRedirect(c: Context, next: Next) {
  const url = new URL(c.req.url);
  const pathname = url.pathname;
  const country = c.req.raw.headers.get('CF-IPCountry') || '';

  // 中国大陆用户重定向到中文版
  const chineseCountries = ['CN', 'HK', 'TW', 'MO'];
  if (chineseCountries.includes(country) && !pathname.startsWith('/zh')) {
    return c.redirect(`https://nanobananamaker.com/zh${pathname}`, 302);
  }

  // 其他地区重定向到英文版（如果访问根路径）
  if (pathname === '/' && !chineseCountries.includes(country)) {
    return c.redirect('https://nanobananamaker.com/en', 302);
  }

  await next();
}
```

**关键文件**：
- `backend/src/index.ts`（注册中间件）
- `backend/src/middleware/geoRedirect.ts`（新建）

export const formatCurrency = (amount: number, lang: string) => {
  const locale = lang === 'zh' ? 'zh-CN' : 'en-US';
  const currency = lang === 'zh' ? 'CNY' : 'USD';

  return new Intl.NumberFormat(locale, {

## 常见问题

### Q: 页面没有被索引怎么办？

```
1. 检查 robots.txt 是否允许爬取
2. 在 GSC 中使用"URL 检查"工具
3. 手动请求编入索引
4. 检查是否有 noindex 标签
5. 确保页面有内部链接指向
```

### Q: 关键词排名下降怎么办？

```
1. 检查竞争对手是否有新内容
2. 更新和丰富页面内容
3. 增加内部链接
4. 检查技术问题 (加载速度等)
5. 获取更多外部链接
```

### Q: 如何提高 CTR？

```
1. 优化标题标签 (包含数字、情感词)
2. 优化 meta description (包含行动召唤)
3. 使用结构化数据获取富媒体摘要
4. 测试不同的标题格式
```
